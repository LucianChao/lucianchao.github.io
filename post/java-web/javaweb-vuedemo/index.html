完成省、市、区的联动展示#

要求页面加载完毕后，默认加载并展示出第一个省、第一个市、第一个区。

这里给出后端服务器数据地址

省：https://web-server.itheima.net/province
市：https://web-server.itheima.net/city?pid=xxx
区：https://web-server.itheima.net/area?cid=xxx

第一版#

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vue3-案例1</title>
    <style>
      #center {
        margin-bottom: 20px;
      }
      input,
      select {
        width: 17%;
        padding: 10px;
        margin-right: 30px;
        border: 1px solid #ccc;
        border-radius: 4px;
      }
    </style>
  </head>
  <body>
    <div id="app">
      <div id="center">
        省：
        <select v-model="province">
          <option v-for="p in provinces" :key="p.id" :value="p.id">
            {{p.name}}
          </option>
        </select>
        市：
        <select v-model="city">
          <option v-for="c in cities" :key="c.id" :value="c.id">
            {{c.name}}
          </option>
        </select>
        区：
        <select v-model="area">
          <option v-for="a in areas" :key="a.id" :value="a.id">
            {{a.name}}
          </option>
        </select>
      </div>
    </div>
    <script src="https://unpkg.com/axios/dist/axios.min.js"></script>
    <script type="module">
      import { createApp } from "https://unpkg.com/vue@3/dist/vue.esm-browser.js";
      createApp({
        data() {
          return {
            province: "",
            city: "",
            area: "",

            provinces: [],
            cities: [],
            areas: [],
          };
        },
        methods: {
          search() {
            // 动态加载省市区
            // 1.获取所有的省 https://web-server.itheima.net/province
            axios
              .get("https://web-server.itheima.net/province")
              .then((result) => {
                console.log(result.data);
                this.provinces = result.data.data;
                this.province = this.provinces[0].id;

                // 为什么要写在获取省的里面？
                // 因为这是一个异步函数，有时候可能pid还没赋值，也就是还没获取到${this.province}的值
                // 已经走向下一步了，所以写在里面，确保代码已经给${this.province}赋值之后再进行下一步
                // 2.获取第一个省对应的市 https://web-server.itheima.net/city?pid=xxx
                axios
                  .get(
                    `https://web-server.itheima.net/city?pid=${this.province}`
                  )
                  .then((result) => {
                    this.cities = result.data.data;
                    this.city = this.cities[0].id;

                    // 获取第一个市对应的区
                    axios
                      .get(
                        `https://web-server.itheima.net/area?cid=${this.city}`
                      )
                      .then((result) => {
                        this.areas = result.data.data;
                        this.area = this.areas[0].id;
                      });
                  });
              });
          },
        },
        mounted() {
          this.search();
        },
      }).mount("#app");
    </script>
  </body>
</html>

可用很轻易的发现，这段代码的可读性很差，函数之间互相嵌套，互相调用

这就是所谓的

回调地狱

信息

回调函数地狱：这种情况发生在需要执行一系列的异步操作，而每个操作都依赖于前一个操作的返回结果，导致代码嵌套过深、可读性差、不便于维护。

async & await#

可以通过async, await来解决回调函数地狱问题。async就是声明一个异步方法，await是用来等待异步任务执行。

methods: {
    async search() {
        const presult = await axios.get("https://web-server.itheima.net/province");
        this.provinces = result.data.data;
        this.province = this.provinces[0].id;

        const presult = await axios.get("https://web-server.itheima.net/city?pid=${this.province}");
        this.provinces = result.data.data;
        this.province = this.provinces[0].id;

        const presult = await axios.get("https://web-server.itheima.net/area?cid=${this.city}");
        this.provinces = result.data.data;
        this.province = this.provinces[0].id;
    }
}

这样的代码： 可读性强、便于维护

信息

注意：await关键字只在async函数内有效，await关键字取代then函数，等待获取请求成功的结果值。
