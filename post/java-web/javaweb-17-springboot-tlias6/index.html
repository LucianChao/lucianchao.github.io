  3. 创建Service层接口以及实现类和Mapper层（省略接口层代码）
    • ClazzServiceImpl.Java 
      package com.javaweb.service.impl;

import com.github.pagehelper.Page;
import com.github.pagehelper.PageHelper;
import com.javaweb.entity.Clazz;
import com.javaweb.entity.ClazzQueryParam;
import com.javaweb.entity.PageBean;
import com.javaweb.mapper.ClazzMapper;
import com.javaweb.service.ClazzService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class ClazzServiceImpl implements ClazzService {
    @Autowired
    private ClazzMapper clazzMapper;
    @Override
    public PageBean page(ClazzQueryParam param) {
        //1.用pageHelper设置分页参数
        PageHelper.startPage(param.getPage(), param.getPageSize());
        //2.调用mapper方法
        List<Clazz> clazzList = clazzMapper.list(param);
        Page page = (Page) clazzList;
        //3.封装PageBean对象并返回
        return new PageBean(page.getTotal(), page.getResult());
    }
}
    • ClazzMapper.java 
      package com.javaweb.mapper;

import com.javaweb.entity.Clazz;
import com.javaweb.entity.ClazzQueryParam;
import org.apache.ibatis.annotations.Mapper;

import java.util.List;

@Mapper
public interface ClazzMapper {
    List<Clazz> list(ClazzQueryParam param);
}
    • ClazzMapper.xml 
      <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
<mapper namespace="com.javaweb.mapper.ClazzMapper">
    <!--条件查询-->
    <select id="list" resultType="com.javaweb.entity.Clazz">
        select c.id,
        c.name,
        c.room,
        c.begin_date,
        c.end_date,
        c.master_id,
        c.create_time,
        c.update_time,
        e.name,
        c.create_time,
        c.update_time
        from clazz c
        left join emp e on c.master_id = e.id
        <where>
            <if test="name != null and name != ''">
                c.name like concat('%',#{name},'%')
            </if>
            <if test="begin != null and end != null">
                and c.end_date between #{begin} and #{end}
            </if>
        </where>
        order by c.update_time desc
    </select>
</mapper>

再次进行联调

如我所料，这种错误本可以避免，但有时就是难免出现这种粗心大意😓。

一开始我以为subject字段需要手动补齐，现在看来我是错的，这波很猪鼻
看了一下clazz表，其实是有这个字段的，没查询，手动补上果然有了。

但此时我又开始思考，一开始我的条件查询语句是严格按照接口文档中的响应数据来的
如果我直接select c.* e.name masterName from ......会怎么样呢？这样只需补充一个masterName就能完成全部数据的查询，多出来的数据前端还会接收吗？

让我们试试看吧，如果成功，这会省去很多工作

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
<mapper namespace="com.javaweb.mapper.ClazzMapper">
    <!--条件查询-->
    <select id="page" resultType="com.javaweb.entity.Clazz">
        select
        c.*,
        e.name masterName
        from clazz c
        left join emp e on c.master_id = e.id
        <where>
            <if test="name != null and name != ''">
                c.name like concat('%',#{name},'%')
            </if>
            <if test="begin != null and end != null">
                and c.end_date between #{begin} and #{end}
            </if>
        </where>
        order by c.update_time desc
    </select>
</mapper>

成功了！我查看了之前员工管理的条件查询，也是这样的，但是我不明白为什么，多余出来的字段去哪里了？前端直接不接收吗？

问了一下gpt
感兴趣的朋友可以看看这个回答

这其实是MyBatis的作用之一，它的工作正是查询数据并映射到对象，且它是 按字段匹配的 。
我们定义了返回类型为Clazz，所以任何Clazz中没有的字段，它都会直接丢弃。
这一切都是在Mapper层就完成的，所以前端看不到多余的字段。
真强大！真牛逼！真好用！👍
原来如此，很有收获。这正加强了我们对MyBatis的了解。

接下来我们来看最后一个字段status，这个字段是clazz表里没有的，所以我觉得要在Service层进行手动补齐了，根据它的开课时间、结课时间来进行逻辑判断，试试吧！

  • ClazzServiceImpl.java 
    @Override
public PageBean page(ClazzQueryParam param) {
    //1.用pageHelper设置分页参数
    PageHelper.startPage(param.getPage(), param.getPageSize());
    //2.调用mapper方法
    List<Clazz> clazzList = clazzMapper.list(param);
    LocalDate now = LocalDate.now();
    for (Clazz clazz : clazzList) {
        if (clazz.getBeginDate().isAfter(now)) {
            clazz.setStatus("未开班");
        } else if (clazz.getBeginDate().isBefore(now) && clazz.getEndDate().isAfter(now)) {
            clazz.setStatus("已开班");
        } else {
            clazz.setStatus("已结课");
        }
    }
    Page page = (Page) clazzList;
    //3.封装PageBean对象并返回
    return new PageBean(page.getTotal(), page.getResult());
}

新增班级#

分析#

  1. 阅读文档时发现请求参数中有masterId，所以这里应该有一个对班主任名字的请求，对员工的请求应该在EmpMapper中完成，保证规范性

  2. 查看请求列表，可以发现有一个list请求，响应失败

    分析这个请求，可以看见这个请求正是对emp表的list查询
    所以我们可以合理推测，前端页面的 班主任名称，就是通过这个请求获得的，然后根据我们选择的班主任来获取masterId
    所以我们要先完善这个查询班主任的功能。
    这里我觉得有两种可能，第一种是直接查询全部员工，然后从中获取班主任，第二种是只查询班主任。 通过查阅接口文档——查询全部员工发现，这里员工管理还有一个未完成的功能就是查询全部员工，可以断定，这里应该是对全部员工进行查询。
