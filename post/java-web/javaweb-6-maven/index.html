Maven#

什么是Maven？#

Maven是一款用于管理和构建Java项目的工具，是apache旗下的一个开源项目。

信息

Apache 软件基金会，成立于1999年7月，是目前世界上最大的最受欢迎的开源软件基金会，也是一个专门为支持开源项目而生的非盈利组织。
开源项目：https://www.apache.org/index.html#project-list

Maven的作用#

  1. 依赖管理 方便快捷的管理项目依赖的资源（ jar包）
  2. 项目构建 标准化的跨平台（Linux、Windows、Maxos）的自动化项目构建方式
  3. 统一项目构建 提供标准、统一的项目结构

Maven概述#

介绍#

  • Apache Maven是一个项目管理和构建工具，它基于项目对象模型( POM——Project Object Model )的概念，通过一小段描述信息来管理项目的构建
  • 作用：
    • 方便的依赖管理
    • 统一的项目结构
    • 标准的项目构建流程
  • 官网：https://maven.apache.org/

仓库：用于存储资源，管理各州jar包

  • 本地仓库：自己计算机上的资源目录
  • 中央仓库：由Maven团队维护的全球唯一的仓库。仓库地址：https://repo1.maven.org/maven2/
  • 远程仓库（私服）：一般由公司团队搭建的私有仓库

安装#

  • 安装步骤：
    1. 解压 apache-maven-3.9.11-bin.zip
    2. 配置本地仓库：修改 conf/settings.xml中的 <localRepository>为一个指定目录
       <localRepository>E:\Devlop\Tools\apache-maven-3.9.11\repo</localRepository>
    1. 配置阿里云私服：修改 conf/settings.xml 中的<mirrors>标签，为其添加如下子标签：
    <!-- 新版本 -->
<mirror>
   <id>aliyunmaven</id>
   <mirrorOf>*</mirrorOf>
   <name>阿里云公共仓库</name>
   <url>https://maven.ailiyun.com/respsitory/public</url>
</mirror>

<!-- 旧版本 -->
<mirrors>
   <id>alimaven</id>
   <name>aliyun maven</name>
   <url>http://maven.aliyun.com.nexus/content/groups/public/</url>
   <mirrorOf>central</mirrorOf>
</mirrors>
    1. 配置JDK版本：修改 conf/settings.xml 中的profile环境为JDK21（可以不配）
    2. 配置环境变量：MAVEN_HOME为maven的解压目录，并将其bin目录加入PATH环境

配置Maven环境（全局）#

打开IDEA主页面，选择：自定义 → 所有设置 构建、执行、部署 → 构建工具 → Maven

然后分别配置：Maven主路径、用户设置文件、本地仓库。 用户设置文件需要勾选 重写然后选择自己的 settings.xml
如果你的settings.xml中的<localRepository>标签配置正确，本地仓库无需重写，会自动生成。 亦或者你想要手动选择另外一个仓库，也可以勾选重写然后自己选择。 这里给出我的配置以供参考

Maven坐标#

  • 什么是坐标？
    • Maven项目中的坐标是 资源（jar）的唯一标识，通过该坐标可以唯一定位资源位置
    • 使用坐标来 定义项目 或 引入项目中需要的依赖。
  • Maven坐标的主要组成
    • groupId：定义当前Maven项目隶属组织名称（通常是域名反写，例如：com.itheima）
    • artifactId：定义当前Maven项目名称（通常是模块名称，例如order-service、goods-service）
    • version：定义当前项目版本号
      • SNAPSHOW：功能不稳定，尚处于开发中的版本，即 快照版本
      • RELEASE：功能趋于稳定，当前停止更新，可以用于发行的版本

依赖管理#

依赖配置#

  • 依赖：指当前项目运行所需要的jar包，一个项目可以引入多个依赖。
  • 配置：
    1. 在 pom.xml 中编写 <dependencies>标签
    2. 在<dependencies>标签中，使用<dependency>引入坐标
    3. 定义坐标的 groupId, artifactId, version
    4. 点击刷新按钮，引入最新加入的标签

信息

  • 如果引入的依赖在本地仓库不存在，将会连接远程仓库/中央仓库，然后下载依赖。（这个过程会比较耗时，请耐心等待）
  • 如果不知道依赖的坐标信息，可以到 https://mvnrepository.com/ 中搜索

依赖传递#

  • 依赖具有传递性
    • 直接依赖：在当前项目中通过配置建立的依赖关系
    • 间接依赖：被依赖的资源如果依赖其他资源，当前项目间接依赖其他资源

看完这些，我们可以思考一个问题。
上面的示例只有三个模块，关系非常简单。

但在真实开发中，我们往往会使用大量第三方库，而每个库又会继续依赖其他库。随着依赖链越来越长，就很容易出现问题。

  1. 引入了本不需要的依赖
    某些依赖传递你根本用不到，但它被“强制”带进了项目中。这些依赖不仅会让项目臃肿不堪，还可能导致安全风险或版本冲突。
  2. 版本冲突（最典型的问题）
    比如：
    • A依赖B（B依赖commons-io 2.6）
    • 同时A又依赖D（D依赖commons-io 2.4）
      这样两个版本同时出现就会引发不兼容的问题。
  3. 无法控制依赖来源

为了解决这些“强制性依赖带来的副作用”，Maven提供了一个非常重要的机制：依赖排除（Dependency Exclusion）

排除依赖#

指主动断开依赖的资源，被排除的资源无需指定版本
