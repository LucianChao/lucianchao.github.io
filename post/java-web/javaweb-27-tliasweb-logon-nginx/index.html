登录实现#

步骤：

  • 为“登录”按钮绑定事件，点击登录按钮发送异步请求到服务端，执行登录操作。
  • 为“取消”按钮绑定事件，点击取消按钮，清空登录表单数据。
  • index.vue 
    <script setup lang="ts">
let loginForm = ref<LoginEmp>({ username: '', password: '' })
// 登录
const login = async () => {
  // 发起异步请求--登录操作
  const result = await loginApi(loginForm.value)
  if (result.code) {
    ElMessage.success('登录成功')
    // 登录成功，跳转到首页-----通过路由进行跳转
    router.push('/')
  } else {
    ElMessage.error(result.msg)
  }
}

// 重置
const clean = () => {
  // 清理表单数据
  loginForm.value = { username: '', password: '' }
}
</script>
<template>

          <el-button class="button" type="primary" @click="clean">重置</el-button>
        </el-form-item>
      </el-form>
    </div>
  </div>
</template>

令牌存储#

在登录成功后，将令牌等信息存储起来。在后续的请求中，再将令牌取出来，携带到服务端。

提示

、 如果在项目的多个组件中，要共享数据，可以使用Vue3中提供的状态管理库Pinia。

Pinia#

  • Pinia是Vue3的存储库（状态管理工具），它允许跨组件/页面共享状态。(https://pinia.vuejs.org/zh/)
  • Store是保存状态和业务逻辑的实体，承载着全局状态。(有点像一个永远存在的组件，每个组件都可以读取、写入)

Pinia-定义Store#

  • 定义存储登录信息的Store
    • src\stores\counter.ts 
      import { ref } from 'vue'
import { defineStore } from 'pinia'
import { LoginInfo } from '@/api/model/model'

// Store对象--通过defineStore创建--用来管理所有的共享状态
// 第一个参数==>counter：一个唯一标识
// 第二个参数==>定义函数
export const useCounterStore = defineStore('loginEmpInfo', () => {
  // 1.定义一个对象--存储所有用户登录信息
  const loginEmp = ref<LoginInfo>({})

  // 2.存储用户登录信息---loginEMmp对象数据
  const setLoginEmp = (loginInfo: LoginInfo) => {
    loginEmp.value = loginInfo
  }

  // 3.获取用户登录信息---loginEMmp对象数据
  const getLonginEmp = () => {
    return loginEmp.value
  }

  // 4.清除用户登录信息---loginEMmp对象数据
  const removeLoginEmp = () => {
    loginEmp.value = {}
  }

  return { loginEmp, setLoginEmp, getLonginEmp, removeLoginEmp }
})

Pinia-使用Store#

  • 在登录完成后，需要使用定义的Store，往state中存储登录用户信息
    步骤：
  1. 登录成功后，往Pinia中存入令牌信息
  2. 在后续的每一次Ajax异步请求时，从Pinia中获取令牌，在请求头中携带到服务端。可以手段在每一个请求函数中添加，但很繁琐；也可以使用axios拦截器
    • src\utils\request.ts 
      // 添加请求拦截器
request.interceptors.request.use(
  function (config) {
    // 发起请求前
    // 将令牌存入到请求头中
    const loginEmp = store.getLonginEmp()
    if (loginEmp && loginEmp.token) {
      config.headers['token'] = loginEmp.token
    }

    return config
  },
  function (error) {
    // 请求错误处理
    return Promise.reject(error)
  },
)

功能完善#

功能1#

  • 目前，即使用户未登录的情况下访问服务器，服务器会响应401状态码，但是前端并不会跳转到登录页面。

功能2#

  • 问题：页面刷新之后，pinia中存储的令牌数据，就消失了，再次请求就获取不到令牌数据了。
  • 原因：页面刷新，原来的Vue实例卸载，Pinia的Store是挂载在Vue实例上的，故刷新后原有的数据也就丢失了。
  • 方案：Pinia持久化（基于pinia-plugin-persistedstate）。

    提示

    确保已经安装pinia-plugin-persistedstate，若没有安装可以运行npm i pinia-plugin-persistedstate安装插件

  • main.ts 
    // 导入Pinia插件（持久化）
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'
// 创建pinia实例
const pinia=createPinia()
// 为pinia挂载插件
pinia.use(piniaPluginPersistedstate)
//应用pinia、router、element plus插件
app.use(pinia)
app.use(router)
app.use(ElementPlus,{locale:zhCn})

//挂载应用
app.mount('#app')
  • src\stores\loginEmp.ts 
    import { ref } from 'vue'
import { defineStore } from 'pinia'
import type { LoginInfo } from '@/api/model/model'

// Store对象--通过defineStore创建--用来管理所有的共享状态
// 第一个参数==>counter：一个唯一标识
// 第二个参数==>定义函数
// 第三个参数==>{ persist: true }声明要持久化
export const useLoginEmpStore = defineStore(
  'loginEmpInfo',
  () => {
    // 1.定义一个对象--存储所有用户登录信息
    const loginEmp = ref<LoginInfo>({})

    // 2.存储用户登录信息---loginEMmp对象数据
    const setLoginEmp = (loginInfo: LoginInfo) => {
      loginEmp.value = loginInfo
    }

    // 3.获取用户登录信息---loginEMmp对象数据
    const getLonginEmp = () => {
      return loginEmp.value
    }

    // 4.清除用户登录信息---loginEMmp对象数据
    const removeLoginEmp = () => {
      loginEmp.value = {}
    }

    return { loginEmp, setLoginEmp, getLonginEmp, removeLoginEmp }
  },
  { persist: true },
)

  • 在nginx的安装目录中，执行sbin目录下的nginx命令，启动mginx服务：sbin/naginx

后端项目部署#

  • 执行maven的父工程中的package生命周期，对项目进行打包【打包之前，先连接上服务器数据库，先测试通过】。
  • 在linux服务器的/usr/local目录下，创建一个目录app，将jar包上传到服务器的/usr/local/app目录中。
  • 然后在命令行执行命令，运行jar包：java-jar xxxxxx.jar
  • 上述执行运行jar包之后，会占用前台窗口，窗口关闭服务也就停了。可以使用nohup指令，后台运行服务，执行指令：nohup java -jar xxxxxx.jar &> tlias.log & 查看进程：ps-ef丨grep xxxx 杀死进程：kiLl -9 xxxx

信息

Linux中的特殊符号：

  • |：管道符。将前面命令的输出，作为后面命令的输入。如：ps-ef | grep java
  • >与>>：重定向符号。将前面的文本内容，输出到后面的文件中（>：覆盖重定向，»：追加重定向）。如：echo 'HeLlo Linux' > 1.log
