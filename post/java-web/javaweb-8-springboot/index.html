Spring框架#

Spring 是一个轻量级、开源的 Java 企业级开发框架，最初由 Rod Johnson 在 2003 年推出。它的核心目标是：

  • 简化 Java 企业级开发
  • 降低代码耦合度
  • 提供一致、便利的编程模型

随着 Java EE 的复杂度不断增加，Spring 通过提供 IoC（控制反转）和 AOP（面向切面编程）两大核心机制，使开发者能够更高效、更灵活地构建应用程序。

  • 官网： https://spring.io
  • Spring发展到今天已经形成了一种开发生态圈，Spring提供了若干个子项目，每个项目用于完成特定的功能。
    • Spring Framework：IoC、AOP 的核心功能
    • Spring MVC：处理 Web 请求的 MVC 框架
    • Spring Boot：快速构建独立、可运行的 Spring 应用
    • Spring Cloud：微服务治理与分布式架构
    • Spring Data：统一的数据访问层框架

  2. 定义HelloController类，添加方法hello，并添加注解
    • src/main/java/com/javaweb/HelloController.java 
      package com.javaweb;

import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

/**
 * 请求处理类
 */
@RestController//标识当前类是一个请求处理类
public class HelloController {
    @RequestMapping("/hello")//用来设置请求路径
    public String hello() {
//        在控制台输出
        System.out.println("Hello......World");
//        方法返回
        return "hello world";
    }
}

入门程序解析#

在上一章的《Tomcat和Servlet》中，我们启动一个Web项目，需要先配置Tomcat

为什么在SpringBoot中一个main方法就将web应用启动了？？ 因为起步依赖中封装了Tomcat 起步依赖：

  • spring-boot-starter-web：包含了web应用开发所需的常见依赖
  • spring-boot-starter-test：包含了单元测试所需要的常见依赖
  • 官方提供的starter：https://docs.spring.io/spring-boot/docs/3.1.3/reference/htmlsingle/#using.build-systems.starters

Tomcat是一个Servlet容器，为什么可用运行我们编写的HelloController？

在SpringBoot进行web程序开发时，底层提供了一个非常核心的DispatchrServlet，请求到达DispatchrServlet之后，会根据请求路径，将请求转给我们定义的Controller程序，最终由Controller程序进行逻辑的处理。
