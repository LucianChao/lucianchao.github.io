前言#

前几章我们讲了类、对象、继承、多态，以及 static 与 final。
现在我们要介绍一个更灵活、更优雅的工具：内部类（Inner Class）。
内部类的意思很简单：在一个类里再定义一个类。
你可能会问：“为什么要这么做？”

  • 当某个类只会在另一个类中使用时，把它放在外部类里更清晰；
  • 内部类可以访问外部类的成员，即便是 private 也可以；
  • 匿名内部类可以快速实现接口或抽象类，让代码更紧凑。

本章，我们会详细讲解 四种内部类：

  1. 成员内部类（Member Inner Class）
  2. 静态内部类（Static Nested Class）
  3. 局部内部类（Local Inner Class）
  4. 匿名内部类（Anonymous Inner Class）

内部类（Inner Class）#

1.什么是内部类#

内部类（Inner Class） 是定义在另一个类内部的类。
它能访问外部类的所有成员（包括 private），
而外部类也能通过实例化访问内部类对象。

class Outer {
    private String msg = "Hello Inner Class";

    class Inner {
        void show() {
            System.out.println(msg);
        }
    }

    void test() {
        Inner in = new Inner();
        in.show();
    }
}

public class Demo {
    public static void main(String[] args) {
        new Outer().test();
    }
}

输出：

Hello Inner Class

2.内部类的类型#

类型 定义位置 是否有名字 是否能访问外部类成员 是否依附外部对象
成员内部类 类中，方法外 有 ✅ 可以 ✅ 必须依附外部对象
局部内部类 方法中 有 ✅ 可以 ✅ 必须依附外部对象
匿名内部类 方法中 ❌ 无 ✅ 可以 ✅ 必须依附外部对象
静态内部类 类中(用 static 修饰) 有 ❌ 不可直接访问非静态成员 ❌ 不依附外部对象

内部类的核心优势：

  • 提高封装性；
  • 灵活访问外部类成员；
  • 便于组织复杂逻辑。

3.成员内部类( Member Inner Class)#

定义在外部类中、方法外的类。
成员内部类就像外部类的一个成员变量，和外部类对象绑定在一起。

class Outer {
    private String msg = "Hello from Outer";

    class Inner {
        void show() {
            System.out.println(msg); // 可以访问外部类私有成员
        }
    }
}

public class Test {
    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner(); // 创建内部类对象
        inner.show(); // 输出：Hello from Outer
    }
}

成员内部类的特点：

  • 可以直接访问外部类的所有成员，包括 private；
  • 内部类实例必须依附于外部类对象。

4.局部内部类( Local Inner Class)#

局部内部类定义在方法内部，仅在方法作用域内可见。

class Outer {
    void greet() {
        final String message = "Hello from Local";

        class Inner {
            void show() {
                System.out.println(message);
            }
        }

        Inner inner = new Inner();
        inner.show();
    }
}

public class Test {
    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.greet(); // 输出：Hello from Local
    }
}

局部内部类特点：

  • 只能在方法内使用；
  • 可以访问方法内的 final 或 effectively final 变量；
  • 常用于对某段逻辑进行封装。

5.静态内部类( Static Nested Class)#

静态内部类类似 static 成员，不依赖外部类对象。

class Outer {
    static String staticMsg = "Hello from Outer Static";

    static class Inner {
        void show() {
            System.out.println(staticMsg);
        }
    }
}

public class Test {
    public static void main(String[] args) {
        Outer.Inner inner = new Outer.Inner(); // 不需要外部类对象
        inner.show(); // 输出：Hello from Outer Static
    }
}

静态内部类特点：

  • 不能访问外部类的非静态成员；
  • 创建不依赖外部类对象，使用灵活；
  • 常用于工具类或单例模式的实现。
  • 适合逻辑相关但不依赖实例”的场景。

6. 匿名内部类（Anonymous Inner Class）#

匿名内部类没有名字，常用于快速实现接口或抽象类。

interface Greeting {
    void sayHello();
}

public class Test {
    public static void main(String[] args) {
        Greeting g = new Greeting() { // 匿名内部类
            public void sayHello() {
                System.out.println("Hello from Anonymous Inner Class");
            }
        };
        g.sayHello(); // 输出：Hello from Anonymous Inner Class
    }
}

匿名内部类特点：

  • 只能使用一次，通常直接赋值给接口或抽象类引用；
  • 代码简洁，但不适合复杂逻辑；
  • 常用于事件监听、回调或临时实现类。

内部类的总结#

类型 成员访问 是否依赖外部对象 用途
成员内部类 可访问外部类所有成员 依赖 封装和逻辑组织
静态内部类 只能访问外部类静态成员 不依赖 工具类、单例模式
局部内部类 可访问方法局部 final 变量 方法内部 封装局部逻辑
匿名内部类 可访问方法局部 final 变量 方法内部 快速实现接口或抽象类
