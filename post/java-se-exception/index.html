世界不会因为错误而崩溃

前言#

写代码的时候，谁都希望程序顺顺利利地跑完。
但理想很丰满，现实很骨感。
比如：

  • 除以0导致程序崩溃；
  • 文件没找到；
  • 用户输入了奇怪的内容；
  • 网络突然断开……

如果我们不处理这些“异常情况”，程序一旦出错就直接终止。
但 Java 的设计哲学是：

“程序不能因为一次错误就崩溃，必须优雅地处理它。” 为了更优雅地应对问题，Java引入了强大的异常机制（Exception）。
它让我们能够： “不让程序死掉，而是有尊严地报错。”

什么是异常（Exception）#

异常指的是程序在运行时发生的、影响正常流程的事件。
比如：

int a = 10;
int b = 0;
System.out.println(a / b); // ❌ ArithmeticException: / by zero

这行代码运行后，程序会抛出一个异常：ArithmeticException，并中断执行。

在Java中，所有异常都继承自 java.lang.Throwable 类：

Throwable
├── Error（严重错误，比如内存溢出）
└── Exception（程序员可以处理的异常）
        ├── RuntimeException（运行时异常）
        └── 非运行时异常（必须处理，否则编译不过）

异常的分类#

类型 示例 是否必须处理
受检异常（Checked Exception） IOException、SQLException ✅ 必须处理
运行时异常（RuntimeException） NullPointerException、ArithmeticException ❌ 可选处理
错误（Error） OutOfMemoryError、StackOverflowError 🚫 不可恢复

异常的捕获：try-catch#

我们可以使用 try-catch 结构来捕获并处理异常。

public class Demo {
    public static void main(String[] args) {
        try {
            int a = 10 / 0;
            System.out.println("这行不会被执行");
        } catch (ArithmeticException e) {
            System.out.println("出错啦：" + e.getMessage());
        }
        System.out.println("程序继续执行...");
    }
}

输出：

出错啦：/ by zero
程序继续执行...

说明：

  • try 块中放可能出错的代码；
  • catch 块中写处理异常的逻辑；
  • 出现异常时，程序不会崩溃，会转入 catch 块执行。

多个 catch 捕获不同异常#

你可以为不同类型的异常设置不同的处理方式。

try {
    int[] arr = new int[3];
    arr[5] = 10; // ArrayIndexOutOfBoundsException
} catch (ArithmeticException e) {
    System.out.println("数学异常！");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("数组越界啦！");
} catch (Exception e) {
    System.out.println("其他异常：" + e.getMessage());
}

注意：

  • catch 顺序很重要！
    越具体的异常要放前面，越通用的放后面。
    否则会报编译错误。

finally：无论如何都要执行的块#

有些代码不管是否发生异常，都必须执行，比如关闭文件、释放资源。 这就用到了 finally：

try {
    System.out.println("打开文件");
    int a = 10 / 0;
} catch (Exception e) {
    System.out.println("发生异常：" + e);
} finally {
    System.out.println("关闭文件");
}

输出：

打开文件
发生异常：java.lang.ArithmeticException: / by zero
关闭文件

即使中途出错，finally 仍然会执行。

抛出异常：throw 与 throws#

有时，我们不想在当前方法处理异常，而是“甩锅”给调用者。

使用 throw 主动抛出异常#

public class Demo {
    public static void checkAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("年龄不能为负数！");
        }
        System.out.println("年龄合法：" + age);
    }

    public static void main(String[] args) {
        checkAge(-3);
    }
}

输出：

Exception in thread "main" java.lang.IllegalArgumentException: 年龄不能为负数！

使用 throws 声明方法可能抛出的异常#

public static void readFile(String path) throws IOException {
    FileReader fr = new FileReader(path); // 文件可能不存在
}

调用者必须处理：

try {
    readFile("test.txt");
} catch (IOException e) {
    e.printStackTrace();
}

自定义异常#

当系统异常不够表达你的业务含义时，你可以自己定义异常类：

class MyException extends Exception {
    public MyException(String msg) {
        super(msg);
    }
}

使用：

public static void main(String[] args) {
    try {
        throw new MyException("自定义异常出现啦！");
    } catch (MyException e) {
        System.out.println(e.getMessage());
    }
}

异常处理的最佳实践#

  1. 不要无脑 catch (Exception e) 捕获所有异常容易掩盖问题，调试困难。

  2. 日志比打印更重要 在真实项目中，要记录异常日志，而不是简单 System.out.println()。

  3. 不要滥用异常做逻辑判断 异常是用于“异常情况”，不是 if-else 的替代品。

  4. 自定义异常要表达语义 比如：UserNotFoundException、InvalidPasswordException。

常见异常类型及处理技巧#

Java 中的异常分为 编译时异常（checked） 和 运行时异常（unchecked）。

常见运行时异常（RuntimeException）#

  1. NullPointerException

发生原因：对 null 对象调用方法或访问属性。

String s = null;
try {
    System.out.println(s.length());
} catch (NullPointerException e) {
    System.out.println("对象为空，不能调用方法！");
}
  1. ArrayIndexOutOfBoundsException

发生原因：访问数组下标越界。

int[] arr = {1, 2, 3};
try {
    System.out.println(arr[5]);
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("数组越界啦！");
}
  1. ClassCastException

发生原因：类型转换错误。

Object obj = "Hello";
try {
    Integer num = (Integer) obj;
} catch (ClassCastException e) {
    System.out.println("类型转换错误！");
}
  1. ArithmeticException

发生原因：算术运算异常（如除以零）。

try {
    int x = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("除以零错误！");
}

常见编译时异常（Checked Exception）#

  1. IOException（输入输出异常）
import java.io.*;
public class TestIO {
    public static void main(String[] args) {
        try {
            FileReader fr = new FileReader("nofile.txt");
        } catch (IOException e) {
            System.out.println("文件不存在或读取错误！");
        }
    }
}
  1. SQLException（数据库操作异常）
import java.sql.*;
try {
    Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/test", "root", "123");
} catch (SQLException e) {
    System.out.println("数据库连接失败！");
}

异常处理技巧#

  • 先捕获具体异常，再捕获父类异常
try {
    // 代码
} catch (FileNotFoundException e) {
    // 先处理文件未找到异常
} catch (IOException e) {
    // 再处理其他IO异常
}
  • 不要捕获过宽的异常
    捕获 Exception 会掩盖错误，调试困难。
  • 使用 finally 做资源释放
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
} finally {
    try {
        if (br != null) br.close();
    } catch (IOException e) {
        e.printStackTrace();
    }
}

异常链( Exception Chaining)#

异常链是指一个异常被另一个异常包裹，用于保留原始异常信息。
在方法内部捕获异常时，如果想抛给上层，但仍保留原异常，可使用构造方法 Throwable(String message, Throwable cause)。

class ExceptionChainDemo {
    static void methodA() throws Exception {
        try {
            int x = 10 / 0;
        } catch (ArithmeticException e) {
            // 将原异常封装后抛出
            throw new Exception("方法A执行失败", e);
        }
    }

    public static void main(String[] args) {
        try {
            methodA();
        } catch (Exception e) {
            System.out.println(e.getMessage());         // 输出："方法A执行失败"
            System.out.println(e.getCause());           // 输出：java.lang.ArithmeticException: / by zero
        }
    }
}

异常链的好处：上层调用者可以知道发生了异常，同时还可以追踪原始异常原因。
