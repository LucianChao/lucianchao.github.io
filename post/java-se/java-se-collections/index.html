集合框架（Collections Framework）#

1. 为什么需要集合#

在前几章中，我们用数组或对象数组来存储多个元素：

String[] arr = new String[10];

但是数组存在一些限制：

  • 长度固定，无法动态扩展；
  • 缺少便捷的方法进行增删查找；
  • 不同类型的数据处理不方便（除非用 Object[]）。 为了解决这些问题，Java 提供了 集合框架（Collections Framework）。

集合框架的核心思想：提供一套通用的数据结构和操作接口，让我们能方便地存储、访问、修改、排序和查找数据.

2.集合的体系框架#

Java 集合框架主要分为 接口 和 实现类：

Collection 接口
├── List 接口（有序、可重复）
│   ├── ArrayList
│   ├── LinkedList
│   └── Vector
├── Set 接口（无序、不可重复）
│   ├── HashSet
│   ├── LinkedHashSet
│   └── TreeSet
└── Queue 接口（队列）
    ├── LinkedList
    └── PriorityQueue

Map 接口（键值对）
├── HashMap
├── LinkedHashMap
└── TreeMap

特点总结：

  • List：有序、允许重复。
  • Set：无序、元素唯一。
  • Queue：先进先出或按优先级处理。
  • Map：存储键值对，键唯一。

注意：Map 不继承自 Collection，但和集合一样，是框架的重要组成部分。

List接口#

List 是最常用的集合接口之一，它的特点是：

  • 元素有顺序（插入顺序）；
  • 可以包含重复的元素；
  • 可以通过索引访问元素。

常见实现类有：

  • ArrayList：基于动态数组，查询快、增删慢；
  • LinkedList：基于双向链表，增删快、查询慢；
  • Vector：线程安全的动态数组（旧类）。

1 ArrayList#

特点#

  • 基于动态数组实现；
  • 支持随机访问；
  • 默认容量为 10，超过自动扩容；
  • 非线程安全（多线程环境需使用 Collections.synchronizedList() 包装）。

常用方法#

import java.util.*;

public class DemoArrayList {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();

        list.add("Apple");          // 添加元素
        list.add("Banana");
        list.add("Orange");
        list.add(1, "Mango");       // 指定位置插入
        System.out.println(list);   // [Apple, Mango, Banana, Orange]

        list.remove("Banana");      // 按值删除
        list.remove(0);             // 按索引删除
        System.out.println(list);   // [Mango, Orange]

        String fruit = list.get(0); // 按索引访问
        System.out.println(fruit);  // Mango

        list.set(1, "Grape");       // 修改指定位置
        System.out.println(list);   // [Mango, Grape]

        System.out.println(list.size());     // 2
        System.out.println(list.contains("Mango")); // true
    }
}

⚠️ 注意：

  • ArrayList 的底层是 Object[] 动态数组。
  • 当元素超过容量时，会创建新数组并复制旧内容。
  • 因此频繁插入或删除中间元素性能较差。

2.LinkedList#

特点：

  • 基于 双向链表 实现；
  • 插入、删除效率高；
  • 适合用作 队列 或 栈；
  • 实现了 List 和 Deque（双端队列）接口。

常用方法

import java.util.*;

public class DemoLinkedList {
    public static void main(String[] args) {
        LinkedList<String> list = new LinkedList<>();

        list.add("A");
        list.add("B");
        list.addFirst("Start");  // 头插入
        list.addLast("End");     // 尾插入
        System.out.println(list); // [Start, A, B, End]

        list.removeFirst();      // 删除第一个
        list.removeLast();       // 删除最后一个
        System.out.println(list); // [A, B]

        System.out.println(list.getFirst()); // 取第一个
        System.out.println(list.getLast());  // 取最后一个
    }
}

💡 适合用在频繁插入、删除操作的场景。
但如果经常随机访问元素（get(i)），性能比 ArrayList 差。

Vector#

特点：

  • 与 ArrayList 类似，但 线程安全；
  • 所有方法都用 synchronized 修饰；
  • 较旧，不推荐新项目中使用。

示例：

import java.util.*;

public class DemoVector {
    public static void main(String[] args) {
        Vector<Integer> v = new Vector<>();

        v.add(10);
        v.add(20);
        v.add(30);

        System.out.println(v.get(1)); // 20
        System.out.println(v.size()); // 3
    }
}

⚠️ 如果你在多线程环境下需要安全的列表，可以考虑：

List<String> safeList = Collections.synchronizedList(new ArrayList<>());

Set接口#

Set 接口代表 不包含重复元素 的集合。
与 List 不同，Set 不保证元素顺序，且每个元素只能出现一次。

常见实现类：

  • HashSet：基于哈希表，元素无序；
  • LinkedHashSet：基于哈希表 + 双向链表，有插入顺序；
  • TreeSet：基于红黑树，元素有序。

1. HashSet#

特点#

  • 不允许重复；
  • 元素无序；
  • 查询、插入、删除都很快（O(1) 期望时间）；
  • 允许存储一个 null 值。

示例#

import java.util.*;

public class DemoHashSet {
    public static void main(String[] args) {
        Set<String> set = new HashSet<>();

        set.add("Apple");
        set.add("Banana");
        set.add("Orange");
        set.add("Apple"); // 重复元素被忽略

        System.out.println(set); // 可能输出 [Orange, Banana, Apple]
        System.out.println(set.contains("Apple")); // true

        set.remove("Banana");
        System.out.println(set.size()); // 2
    }
}

⚠️ 注意：

  • 元素存储无序，因为 HashSet 底层使用 哈希表 (HashMap)。
  • 若要存放自定义对象，必须重写 hashCode() 和 equals()，否则去重会失败。

2. LinkedHashSet#

特点：

  • 与 HashSet 类似；
  • 但通过双向链表 记录插入顺序；
  • 迭代时会按插入顺序输出。

示例：

import java.util.*;

public class DemoLinkedHashSet {
    public static void main(String[] args) {
        Set<String> set = new LinkedHashSet<>();

        set.add("Apple");
        set.add("Banana");
        set.add("Orange");

        System.out.println(set); // [Apple, Banana, Orange]
    }
}

💡 当你希望 去重但保留顺序 时，用 LinkedHashSet。

3. TreeSet#

特点：

  • 元素 自动排序（升序）；
  • 底层基于 红黑树；
  • 不允许 null；
  • 可通过 Comparator 自定义排序规则。 示例：
import java.util.*;

public class DemoTreeSet {
    public static void main(String[] args) {
        Set<Integer> set = new TreeSet<>();

        set.add(30);
        set.add(10);
        set.add(20);

        System.out.println(set); // [10, 20, 30]

        // 自定义排序（按字符串长度）
        Set<String> words = new TreeSet<>(Comparator.comparingInt(String::length));
        words.add("apple");
        words.add("kiwi");
        words.add("banana");
        System.out.println(words); // [kiwi, apple, banana]
    }
}

💡 适用于需要保持排序状态的集合，如排行榜、分数表等。

4. Set小结#

实现类 底层结构 是否有序 是否可重复 适用场景
HashSet 哈希表 否 否 快速查找、去重
LinkedHashSet 哈希表 + 链表 是（插入顺序） 否 保留插入顺序
TreeSet 红黑树 是（自动排序） 否 排序集合

Map接口#

Map 是一种 键值对 (key-value) 形式的集合，每个键（key）唯一，对应一个值（value）。

与 Collection 不同，Map 不是继承自 Collection 接口的，
它有自己的一套存储结构和遍历方式。

常见实现类：

  • HashMap：基于哈希表，存取快；
  • LinkedHashMap：保持插入顺序；
  • TreeMap：按键排序；
  • Hashtable：线程安全（已过时，通常用 ConcurrentHashMap 代替）。

1. HashMap#

特点#

  • 键唯一；
  • 值可重复；
  • 无序；
  • 允许一个 null 键、多个 null 值；
  • 非线程安全。

示例#

import java.util.*;

public class DemoHashMap {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();

        map.put("Apple", 10);
        map.put("Banana", 20);
        map.put("Orange", 30);
        map.put("Apple", 15); // 覆盖原值

        System.out.println(map); // {Orange=30, Apple=15, Banana=20}
        System.out.println(map.get("Apple")); // 15
        System.out.println(map.containsKey("Banana")); // true
        System.out.println(map.containsValue(30)); // true

        map.remove("Orange");
        System.out.println(map.size()); // 2
    }
}

💡 HashMap 是最常用的 Map 实现，底层用 哈希表 + 链表/红黑树。

2.遍历Map的四种方式#

Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);

// 1️⃣ 遍历键
for (String key : map.keySet()) {
    System.out.println("Key: " + key);
}

// 2️⃣ 遍历值
for (Integer value : map.values()) {
    System.out.println("Value: " + value);
}

// 3️⃣ 遍历键值对
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

// 4️⃣ 使用 Lambda
map.forEach((k, v) -> System.out.println(k + "=" + v));

3.LinkedHashMap#

特点:

  • 继承自 HashMap；
  • 保留插入顺序；
  • 支持 “访问顺序” 模式（可用于实现 LRU 缓存）。
import java.util.*;

public class DemoLinkedHashMap {
    public static void main(String[] args) {
        Map<String, Integer> map = new LinkedHashMap<>();

        map.put("A", 1);
        map.put("B", 2);
        map.put("C", 3);

        System.out.println(map); // {A=1, B=2, C=3}
    }
}

💡 用于需要同时保持插入顺序和高效查询的场景，比如缓存或配置文件解析。

4.TreeMap#

特点:

  • 键有序；
  • 底层为 红黑树；
  • 不允许 null 键；
  • 可通过 Comparator 自定义排序。

示例：

import java.util.*;

public class DemoTreeMap {
    public static void main(String[] args) {
        Map<Integer, String> map = new TreeMap<>();

        map.put(3, "C");
        map.put(1, "A");
        map.put(2, "B");

        System.out.println(map); // {1=A, 2=B, 3=C}

        // 自定义排序（键降序）
        Map<Integer, String> desc = new TreeMap<>(Comparator.reverseOrder());
        desc.putAll(map);
        System.out.println(desc); // {3=C, 2=B, 1=A}
    }
}

5. Map小结#

实现类 底层结构 是否有序 是否允许 null 键 是否线程安全 适用场景
HashMap 哈希表 否 ✅ 允许 ❌ 否 最常用的一般场景
LinkedHashMap 哈希表 + 双向链表 ✅ 插入有序 ✅ 允许 ❌ 否 缓存、有序输出
TreeMap 红黑树 ✅ 排序有序 ❌ 不允许 ❌ 否 按键排序
Hashtable 哈希表 否 ❌ 不允许 ✅ 是 老旧同步类（不推荐）

Collections 工具类与集合框架总结#

java.util.Collections 是一个非常实用的工具类，
提供了大量静态方法，用于 操作或返回集合对象，比如：

  • 排序、查找、反转；
  • 创建线程安全集合；
  • 创建只读集合；
  • 批量填充与复制等。

6.1 常用静态方法示例#

排序 sort()#

import java.util.*;

public class DemoSort {
    public static void main(String[] args) {
        List<Integer> list = Arrays.asList(3, 1, 4, 1, 5);
        Collections.sort(list);
        System.out.println(list); // [1, 1, 3, 4, 5]
    }
}

若要按自定义规则排序，可传入 Comparator：

Collections.sort(list, Comparator.reverseOrder());

反转 reverse()#

List<String> list = new ArrayList<>(List.of("A", "B", "C"));
Collections.reverse(list);
System.out.println(list); // [C, B, A]

随机打乱 shuffle()#

List<Integer> list = new ArrayList<>(List.of(1, 2, 3, 4, 5));
Collections.shuffle(list);
System.out.println(list); // 随机顺序

查找最大值/最小值#

List<Integer> list = List.of(10, 30, 20);
System.out.println(Collections.max(list)); // 30
System.out.println(Collections.min(list)); // 10

复制cppy()#

List<String> src = List.of("A", "B", "C");
List<String> dest = new ArrayList<>(Arrays.asList("", "", ""));
Collections.copy(dest, src);
System.out.println(dest); // [A, B, C]

注意：目标列表必须有足够的空间，否则抛 IndexOutOfBoundsException。

填充 fill()#

List<String> list = new ArrayList<>(Arrays.asList("X", "Y", "Z"));
Collections.fill(list, "A");
System.out.println(list); // [A, A, A]

线程安全集合 synchronizedXXX()#

List<Integer> list = new ArrayList<>();
List<Integer> safeList = Collections.synchronizedList(list);

这样可以让普通集合在多线程环境下安全使用。
不过现代开发中，更推荐使用 java.util.concurrent 包下的并发集合（如 ConcurrentHashMap）。

不可变集合 unmodifiableXXX()#

List<String> list = List.of("A", "B", "C");
List<String> unmodifiable = Collections.unmodifiableList(list);

// unmodifiable.add("D"); // 抛出 UnsupportedOperationException

用于防止集合被修改，常见于只读配置或返回安全视图。

小结#

分类 接口 常用实现类 特点
List 有序、可重复 ArrayList / LinkedList 可通过索引访问
Set 无序、不重复 HashSet / TreeSet 用于去重
Queue 队列结构 LinkedList / PriorityQueue 先进先出
Map 键值对映射 HashMap / TreeMap 键唯一
工具类 Collections 排序、同步、只读、查找等操作 静态方法
