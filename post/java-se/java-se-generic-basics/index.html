泛型#

泛型的概念#

在没有泛型之前，集合类可以存放任意类型的对象：

ArrayList list = new ArrayList();
list.add("Hello");
list.add(123);

这样做的问题是：

  • 类型不安全（编译时不检查类型）
  • 取出时必须强制类型转换
  • 容易出错

泛型（Generic）就是为了解决这些问题。
它在编译期就能检查类型安全，减少类型转换。

ArrayList<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123); // 编译错误
String s = list.get(0); // 不需要强制转换

2.泛型的声明和使用#

泛型类#

在类名后加上 <T>，T 表示“类型参数”。

class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

Box<String> b1 = new Box<>();
b1.set("Java");
System.out.println(b1.get());

这里的 T 可以替换成任何合法的类型名，例如 E、K、V、U，取决于场景。

多类型参数#

泛型类可以有多个类型参数：

class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }
    public K getKey() { return key; }
    public V getValue() { return value; }
}

使用：

Pair<String, Integer> student = new Pair<>("Alice", 18);
System.out.println(student.getKey() + " - " + student.getValue());

泛型方法#

泛型不仅可以用在类上，也可以用在方法上：

public class Tool {
    public static <T> void printArray(T[] arr) {
        for (T t : arr) {
            System.out.print(t + " ");
        }
        System.out.println();
    }
}

调用时可以自动推断类型：

Integer[] nums = {1, 2, 3};
String[] strs = {"A", "B"};
Tool.printArray(nums);
Tool.printArray(strs);

泛型接口#

interface Pair<K, V> {
    K getKey();
    V getValue();
}

class KeyValue<K, V> implements Pair<K, V> {
    private K key;
    private V value;
    KeyValue(K key, V value) { this.key = key; this.value = value; }
    public K getKey() { return key; }
    public V getValue() { return value; }
}

示例：

Pair<String, Integer> p = new KeyValue<>("Age", 20);
System.out.println(p.getKey() + ": " + p.getValue());

泛型的类型擦除#

Java 的泛型是伪泛型，在运行时会被类型擦除（Type Erasure）：

  • 编译后所有泛型信息会被去掉；
  • 泛型参数会被替换为 Object 或者限定类型。

例如：

List<String> a = new ArrayList<>();
List<Integer> b = new ArrayList<>();
System.out.println(a.getClass() == b.getClass()); // true

因此在运行时无法区分 List<String> 和 List<Integer>。

泛型通配符与边界限定#

泛型的强类型约束让代码更安全，但有时也会让我们“过于严格”。
比如：

List<Integer> list1 = new ArrayList<>();
List<Double> list2 = new ArrayList<>();

虽然 Integer 和 Double 都是 Number 的子类，但：

void printList(List<Number> list) { ... }

无法接受 List<Integer>，因为 List<Integer> 不是 List<Number> 的子类。

为了解决这种情况，Java 提供了 通配符 ?。

1.无界通配符 ?#

? 表示“某种未知类型”，常用于只读的场景：

public static void printList(List<?> list) {
    for (Object obj : list) {
        System.out.print(obj + " ");
    }
}

调用时可以传入任何类型的列表：

printList(List.of("Java", "C++", "Python"));
printList(List.of(1, 2, 3, 4));

注意： 使用 List<?> 时，不能向其中添加元素（除了 null），因为编译器不知道它到底是什么类型。

2.上界通配符 ? extends T#

表示 某个未知类型是 T 的子类。
典型用途：读取时安全，写入时受限。

public static double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

调用:

List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(2.5, 3.5);
System.out.println(sum(ints));    // ✅ OK
System.out.println(sum(doubles)); // ✅ OK

3.下界通配符 ? super T#

表示 某个未知类型是 T 的父类。 典型用途：写入时安全，读取时受限。

public static void addNumbers(List<? super Integer> list) {
    list.add(10);
    list.add(20);
}

调用：

List<Number> nums = new ArrayList<>();
addNumbers(nums); // ✅ OK

? super Integer 意味着可以安全地向其中加入 Integer 或其子类对象。

4.通配符的比较#

通配符 含义 可读？ 可写？ 常用场景
? 任意类型 ✅ ❌ 只读列表
? extends T T 的子类 ✅ ❌ 上界读取（只读）
? super T T 的父类 ❌（只能读 Object） ✅ 下界写入（只写）

记忆口诀

“extends 读，super 写”
