前言#

在今天这一章，我们将走进一个“神秘”的领域：反射（Reflection）与注解（Annotation）。
这两个概念是 Java 高级编程中非常重要的内容，也是框架（如 Spring、MyBatis）运作的核心机制。
掌握它们，你就能理解 Java 是如何“在运行时”了解自己、修改自己、甚至自动配置的。

反射（Reflection）#

1.什么是反射#

简单来说，反射就是让程序在运行时动态地了解和操作类的信息的能力。
比如说，我们平常写代码时，类名、方法名都是写死的：

Person p = new Person();
p.sayHello();

而有了反射，我们就可以这样写：

Class<?> c = Class.forName("Person");
Object obj = c.getDeclaredConstructor().newInstance();
Method m = c.getMethod("sayHello");
m.invoke(obj);

不用在代码里写 Person 或 sayHello，只要知道它们的名字，就能在程序运行时动态地调用。

2. 获取 Class 对象的三种方式#

在 Java 中，一切反射操作都从 Class 对象开始。 要进行反射，第一步就是拿到某个类的 Class 对象。

// 方法一：通过类名
Class<?> c1 = Person.class;

// 方法二：通过对象
Person p = new Person();
Class<?> c2 = p.getClass();

// 方法三：通过字符串（最常用）
Class<?> c3 = Class.forName("Person");

这三种方式效果相同，但第三种最灵活，因为类名可以从配置文件、用户输入等地方动态传入。

3. 使用反射创建对象#

一旦我们有了 Class 对象，就能创建类的实例：

Person p = (Person) c3.getDeclaredConstructor().newInstance();

getDeclaredConstructor() 获取无参构造函数， newInstance() 表示创建对象 如果有参数构造器，可以这样：

Constructor<?> cons = c3.getConstructor(String.class, int.class);
Person p2 = (Person) cons.newInstance("小明", 18);

4. 反射调用方法#

反射不仅能创建对象，还能调用方法：

Method m = c3.getMethod("sayHello");
m.invoke(p);  // 相当于 p.sayHello();

如果方法有参数:

Method m2 = c3.getMethod("setName", String.class);
m2.invoke(p, "张三");

5. 反射访问属性（字段）#

反射同样可以访问字段（包括私有字段）：

Field f = c3.getDeclaredField("name");
f.setAccessible(true);  // 打开“安全锁”，允许访问 private 字段
f.set(p, "李四");       // 修改字段值
System.out.println(f.get(p));  // 获取字段值

这就是为什么框架（如 Spring）能在运行时自动给对象“注入属性”。

完整示例#

import java.lang.reflect.*;

class Person {
    private String name;
    private int age;

    public Person() {}
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void sayHello() {
        System.out.println("你好，我是 " + name + "，今年 " + age + " 岁。");
    }
}

public class ReflectDemo {
    public static void main(String[] args) throws Exception {
        Class<?> c = Class.forName("Person");

        Constructor<?> cons = c.getConstructor(String.class, int.class);
        Object obj = cons.newInstance("小明", 10);

        Method m = c.getMethod("sayHello");
        m.invoke(obj);
    }
}

输出：

你好，我是 小明，今年 10 岁。

注解（Annotation）#

1.什么是注解#

注解其实是一种“特殊的标记”，它可以写在类、方法、变量等位置，
用来告诉编译器或程序某些“额外信息”。

最常见的注解有：

  • @Override：表示重写父类方法；
  • @Deprecated：表示方法已过时；
  • @SuppressWarnings：忽略警告 例如：
@Override
public String toString() {
    return "这是重写的方法";
}

2.自定义注解#

我们也可以自己定义注解：

@interface MyAnnotation {
    String value();
}

使用：

@MyAnnotation("Hello")
public void test() {}

3.让注解在运行时可用#

要让反射能读取注解内容，需要加上元注解：

import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME) // 在运行时可见
@Target(ElementType.METHOD)         // 用于方法
public @interface MyAnnotation {
    String value();
}

读取注解信息：

Method m = Demo.class.getMethod("test");
MyAnnotation a = m.getAnnotation(MyAnnotation.class);
System.out.println(a.value());

完整示例：

import java.lang.annotation.*;
import java.lang.reflect.*;

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface MyAnnotation {
    String value();
}

class Demo {
    @MyAnnotation("这是一个测试方法")
    public void test() {}
}

public class AnnotationDemo {
    public static void main(String[] args) throws Exception {
        Method m = Demo.class.getMethod("test");
        MyAnnotation a = m.getAnnotation(MyAnnotation.class);
        System.out.println("注解内容：" + a.value());
    }
}

输出：

注解内容：这是一个测试方法

反射与注解的结合#

在框架开发中，反射 + 注解 常常一起使用。
比如：

  • Spring 用注解 @Autowired + 反射注入对象；
  • JUnit 用注解 @Test + 反射执行测试方法 简化版示例：
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface Run {
}

class TestDemo {
    @Run
    public void hello() {
        System.out.println("运行方法 hello()");
    }

    public void noRun() {}
}

public class RunDemo {
    public static void main(String[] args) throws Exception {
        for (Method m : TestDemo.class.getDeclaredMethods()) {
            if (m.isAnnotationPresent(Run.class)) {
                m.invoke(new TestDemo());
            }
        }
    }
}

运行结果：

运行方法 hello()

这就是一个简易版的JUnit测试框架
