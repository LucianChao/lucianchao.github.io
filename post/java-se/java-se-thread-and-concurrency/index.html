多线程与并发#

线程基础#

1.什么是线程#

线程（Thread）是程序执行的最小单位。一个程序至少有一个主线程（main），通过多线程我们可以让程序同时做多件事。 例如：

  • 浏览器一边加载网页、一边播放音乐。
  • 游戏一边渲染画面、一边检测输入。

多线程可以提升程序效率，但也可能带来同步问题（比如两个线程同时修改同一个变量）。

2.创建线程的三种方式#

继承 Thread 类#

class MyThread extends Thread {
    public void run() {
        System.out.println("线程运行中：" + Thread.currentThread().getName());
    }
}

public class Test {
    public static void main(String[] args) {
        MyThread t = new MyThread();
        t.start(); // 启动线程，自动调用 run()
    }
}

注意：

  • start() 才是启动新线程；
  • 直接调用 run() 只是普通方法调用。

实现 Runnable 接口（推荐）#

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable线程运行：" + Thread.currentThread().getName());
    }
}

public class Test {
    public static void main(String[] args) {
        Thread t = new Thread(new MyRunnable());
        t.start();
    }
}

推荐使用这种方式，因为 Java 不支持多继承，而接口方式更灵活。

使用匿名类或 Lambda 表达式（更简洁）

public class Test {
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            System.out.println("Lambda线程运行：" + Thread.currentThread().getName());
        });
        t.start();
    }
}

Thread类的常用方法

方法 说明
start() 启动新线程
run() 线程执行体（不要手动调用）
sleep(ms) 暂停线程指定毫秒
join() 等待该线程执行结束
isAlive() 判断线程是否还在运行
setPriority(int) 设置线程优先级（1~10）
setName(String) 修改线程名字

示例：

public class Demo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                System.out.println("子线程：" + i);
            }
        });
        t.start();
        t.join(); // 等待 t 执行完
        System.out.println("主线程结束");
    }
}

输出：

子线程：0
子线程：1
子线程：2
主线程结束

3.线程的生命周期#

线程从创建到结束，一共有 5 种状态：

  1. 新建（New）
    • 刚创建线程对象，还没有调用 start()。
    Thread t = new Thread(() -> {});
System.out.println(t.getState()); // NEW
  2. 就绪（Runnable）
    • 调用 start() 后，线程进入就绪状态，等待 CPU 调度。
    t.start();
System.out.println(t.getState()); // RUNNABLE（可能显示不同）
  3. 运行（Running）
    • 正在执行 run() 方法。
    • 无法直接强制进入或查看，只有通过日志或打印判断。
  4. 阻塞/等待（Blocked/Waiting）
    • 阻塞：线程试图获取锁但失败，进入阻塞状态。
    • 等待：调用 wait() 或 join()，等待被唤醒。
    synchronized (obj) {
    obj.wait(); // 当前线程进入等待
}
  5. 终止（Terminated）
    • run() 方法执行完毕，线程结束。
    System.out.println(t.getState()); // TERMINATED

新建(New) ---> 就绪(Runnable) ---> 运行(Running)
                      |                |
                 阻塞/等待(Waiting/Blocked)
                      |
                   再次就绪
                      |
                    终止(Terminated)

注意：

  • 就绪 ≠ 运行，就绪线程需要等待 CPU 调度。
  • 阻塞/等待状态可能由 sleep(), join() 或锁争用触发

sleep()与join()的区别#

方法 作用 状态
sleep(ms) 暂停当前线程，不释放锁 就绪/运行切换
join() 等待另一个线程结束 阻塞/等待

示例：

Thread t1 = new Thread(() -> {
    try {
        Thread.sleep(1000);
        System.out.println("t1结束");
    } catch (InterruptedException e) {}
});

Thread t2 = new Thread(() -> {
    try {
        t1.join(); // 等待 t1 结束
        System.out.println("t2开始");
    } catch (InterruptedException e) {}
});

t1.start();
t2.start();

输出顺序保证：

t1结束
t2开始

线程同步与锁#

在多线程环境下，如果多个线程同时访问共享资源，就可能出现 线程安全问题。
解决方案主要是 同步（Synchronization） 和 锁（Lock）。

1. synchronized 关键字#

synchronized 用于 保证同一时间只有一个线程访问某个代码块或方法。

同步方法#

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}
  • increment() 被 synchronized 修饰，保证同一时刻只有一个线程能执行。
  • 适用于 整个方法需要同步 的情况。

同步块#

class Counter {
    private int count = 0;
    private final Object lock = new Object();

    public void increment() {
        synchronized(lock) {
            count++;
        }
    }
}
  • 只锁定代码块，更灵活，避免不必要的性能开销。

注意：

  • synchronized 锁的是对象（Object），不同对象的锁互不影响。
  • 锁定静态方法时，锁对象是 Class 对象。

ReentrantLock（可重入锁）#

比 synchronized 更灵活，支持：

  • 公平锁（按顺序获取锁）
  • 中断响应锁
  • 尝试获取锁（tryLock）
import java.util.concurrent.locks.ReentrantLock;

class Counter {
    private int count = 0;
    private final ReentrantLock lock = new ReentrantLock();

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock(); // 确保释放锁
        }
    }

    public int getCount() {
        return count;
    }
}

对比 synchronized：

  • synchronized 自动释放锁，语法简单
  • ReentrantLock 更灵活，可响应中断或尝试获取锁

线程安全问题示例#

class Counter {
    private int count = 0;

    public void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        Counter c = new Counter();

        Thread t1 = new Thread(() -> {
            for(int i=0; i<1000; i++) c.increment();
        });
        Thread t2 = new Thread(() -> {
            for(int i=0; i<1000; i++) c.increment();
        });

        t1.start();
        t2.start();
        t1.join();
        t2.join();

        System.out.println(c.getCount()); // 可能小于2000，出现线程安全问题
    }
}

解决方法：

  • 使用 synchronized 或 ReentrantLock
  • 使用原子变量 AtomicInteger（更高效）

死锁#

死锁是指 两个或多个线程互相等待对方释放锁，永远无法继续执行。

class A {
    synchronized void methodA(B b) {
        System.out.println("线程A锁住A");
        try { Thread.sleep(100); } catch(Exception e){}
        b.last();
    }
    synchronized void last() {}
}

class B {
    synchronized void methodB(A a) {
        System.out.println("线程B锁住B");
        try { Thread.sleep(100); } catch(Exception e){}
        a.last();
    }
    synchronized void last() {}
}

public class DeadlockTest {
    public static void main(String[] args) {
        A a = new A();
        B b = new B();
        new Thread(() -> a.methodA(b)).start();
        new Thread(() -> b.methodB(a)).start();
    }
}
  • 输出可能停在： 
    线程A锁住A
线程B锁住B
  • 两个线程互相等待对方释放锁，形成死锁。

避免死锁方法：

  1. 统一锁顺序
  2. 尝试获取锁（tryLock）
  3. 尽量减少锁的持有时间

线程池与并发工具类#

在实际开发中，如果频繁创建和销毁线程，系统开销会非常大。
为了解决这个问题，Java 提供了 线程池（Thread Pool），通过 复用线程 来提高性能和响应速度。

1.什么是线程池#

线程池是一种 线程复用机制，通过预先创建若干线程，当任务到来时从池中取出线程执行，执行完毕后再放回池中。
优点：

  • 避免频繁创建销毁线程的开销；
  • 控制线程数量，防止系统资源耗尽；
  • 支持任务排队、延迟执行、定时执行等功能。

2. Executor 框架结构#

Java 的线程池主要由以下几个核心接口组成：

接口 / 类 说明
Executor 最顶层接口，定义了执行任务的基本方法 execute()
ExecutorService 扩展接口，支持提交任务、关闭线程池、获取返回值
ThreadPoolExecutor 真正的线程池实现类
Executors 工具类，用于创建各种常用线程池

3.创建线程池（Executors 工具类）#

import java.util.concurrent.*;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // 创建固定大小的线程池
        ExecutorService pool = Executors.newFixedThreadPool(3);

        for (int i = 1; i <= 5; i++) {
            int taskId = i;
            pool.execute(() -> {
                System.out.println(Thread.currentThread().getName() + " 执行任务 " + taskId);
            });
        }

        pool.shutdown(); // 关闭线程池（不再接收新任务）
    }
}

常见线程池类型：

创建方法 功能
newFixedThreadPool(n) 固定数量线程池
newCachedThreadPool() 按需创建线程，可重用空闲线程
newSingleThreadExecutor() 单线程池，顺序执行任务
newScheduledThreadPool(n) 定时/周期性执行任务

4.带返回值的任务（Callable + Future）#

Runnable 不能返回结果，而 Callable 可以返回值或抛出异常。
配合 Future 获取异步结果：

import java.util.concurrent.*;

public class CallableDemo {
    public static void main(String[] args) throws Exception {
        ExecutorService pool = Executors.newSingleThreadExecutor();

        Callable<Integer> task = () -> {
            System.out.println("正在计算...");
            Thread.sleep(1000);
            return 123;
        };

        Future<Integer> result = pool.submit(task);

        System.out.println("结果：" + result.get()); // 阻塞等待结果
        pool.shutdown();
    }
}

Future.get() 会阻塞线程，直到结果返回。

5.定时任务（ScheduledExecutorService）#

可以替代 Timer，更强大也更安全。

import java.util.concurrent.*;

public class ScheduleDemo {
    public static void main(String[] args) {
        ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

        // 延迟2秒后执行
        scheduler.schedule(() -> System.out.println("延迟任务执行！"), 2, TimeUnit.SECONDS);

        // 每隔3秒重复执行一次
        scheduler.scheduleAtFixedRate(() -> System.out.println("周期任务执行！"),
                                      1, 3, TimeUnit.SECONDS);
    }
}

6.并发工具简介（java.util.concurrent）#

工具类 功能
CountDownLatch 等待多个线程完成后再继续
CyclicBarrier 一组线程互相等待直到全部到达某个点
Semaphore 控制同时访问资源的线程数量
ConcurrentHashMap 线程安全的哈希表
BlockingQueue 支持阻塞操作的队列，常用于生产者-消费者模型

CountDownLatch 示例：

import java.util.concurrent.*;

public class LatchDemo {
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);

        for (int i = 1; i <= 3; i++) {
            new Thread(() -> {
                System.out.println(Thread.currentThread().getName() + " 执行完毕");
                latch.countDown(); // 计数器减1
            }).start();
        }

        latch.await(); // 等待所有线程结束
        System.out.println("所有任务已完成");
    }
}

✅ 小结：

  • 使用线程池提高性能；
  • 使用 Callable + Future 获取结果；
  • ScheduledExecutorService 可执行定时任务；
  • CountDownLatch、Semaphore 等工具可协调线程执行顺序。
