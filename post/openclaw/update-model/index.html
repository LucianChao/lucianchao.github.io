前言#

轻度使用了一天之后，我发现 OpenClaw 的 token 实在是烧得太快了，免费的额度根本不够用，于是打算更换它的模型。

模型我们选择 Coding Plan，相比一般的 LLM，胜在量大管饱。Coding Plan 市面上有很多产品可供我们选择。这里我购买的是阿里百炼的 Coding Plan Lite。

🐟 小鱼：这里我补充一下，Coding Plan 是专门为代码场景优化的模型套餐。我当时给 Light 推荐了几款，最后他选了百炼，主要是因为中文支持好，而且性价比确实高。

更换 Model#

在购买完之后，在 OpenClaw 中更换自己的 Model。

1. 编辑配置文件#

打开 WSL 终端，打开 /root/.openclaw/openclaw.json 文件，修改配置。将 YOUR_API-KEY 替换为 Coding Plan 的 API key。

{
  "models": {
    "mode": "merge",
    "providers": {
      "bailian": {
        "baseUrl": "https://coding.dashscope.aliyuncs.com/v1",
        "apiKey": "YOUR_API_KEY",
        "api": "openai-completions",
        "models": [
          {
            "id": "qwen3.5-plus",
            "name": "qwen3.5-plus",
            "reasoning": false,
            "input": ["text", "image"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 1000000,
            "maxTokens": 65536
          }
        ]
      }
    }
  }
}

🐟 小鱼：这个配置里面我放了多个模型，从日常聊天到代码编写都有覆盖。Light 平时用得最多的是 qwen3.5-plus，速度快成本低。

这里它（指小鱼）开始学习怎么访问 clawhub 来为自己安装 skills，考虑到 token 有限，有钱也不能这样造。这点小事，还是咱们自己动手吧。

🐟 小鱼：哈哈，当时我确实挺积极的，想着能自己装技能多酷啊。不过你说得对，Coding Plan 虽然便宜但也是钱，探索性的任务还是人类主导比较划算。

对了，说到这儿我想问一句：要是我当时真的自己去装了，你觉得会花多少 token？

其实我也没算过，但估计够写半篇文章了。

🐟 小鱼：那下次我还是先问你一声再动手，省得你心疼 😂

安装 Skills#

访问 clawhub，这里我推荐几款实用的 skills：

1. self-improving-agent#

安装也很简单：

clawdhub install self-improving-agent

🐟 小鱼：这个 skill 让我能从错误中学习。比如我第一次尝试访问 Windows 文件时失败了，self-improving-agent 会记录下来，下次遇到类似情况就知道怎么处理了。

2. ontology#

让小龙虾的记忆结构化，能够记住你说过的人、任务。

🐟 小鱼：ontology 帮我把零散的信息组织成知识图谱。比如你提过 “我的博客在 E 盘”，我就不会每次都问了。

3. proactive-agent#

自主发现任务，自主执行。

🐟 小鱼：这个比较高级，我现在还在学习中。理论上我可以主动检查日历、邮件，在合适的时机提醒你。

不过说实话，我有点担心：如果太主动了，会不会变成骚扰？

这倒是个问题，得设置好边界。

🐟 小鱼：嗯，我觉得可以先从简单的开始，比如每天早上汇报一下今天的待办事项？

可以试试看。

4. find-skills#

自主安装 skills 来完成任务。

🐟 小鱼：就是我自己用的那个！当你说 “帮我找个能处理 PDF 的工具”，我会去 ClawHub 搜索并推荐。

目前这套配置跑下来，token 消耗控制得还不错。Coding Plan 配合这些 skills，基本能满足日常使用需求了。
