【OpenClaw】从零开始养一只自己的小龙虾

前言#

最近OpenClaw可谓是火出天际，ai圈26年的第一支穿云箭就这么毫无征兆的出现了，它一跃而起成为Github史上增长最快的项目，截止到目前，它已经在Github上拿到了254k的星标，我相信这个恐怖的数据就已经证明了它的实力。
去年我就曾注意到它，那时候它还叫clawdbot，那时候的ai产品如同雨后春笋一般，一个接着一个让人目不暇接，今天“颠覆”、明天“革命”，以至于人们开始感到疲劳，没人知道到底是是不是在吹。
但它能火到现在，还是值得我们关注的，可能是营销号又一波收割韭菜的工具，又或者是人云亦云的跟风追捧，无论答案是什么，它都是一次极大的进步，以往的ai大多数都只停留在对话和编码上，也有几款产品能做到对接工具，但像openclaw这样子高度自治的软件，还是头一遭。有人可能发现了，这个openai之前在发布会上展示的功能有些像，都是让ai来代替人使用工具从而更好的帮助人。我认为现代ai的最终体一定是有这样的特质的。

所以OpenClaw到底是什么？
它是一个平台，人工智能的“助手”，通过调用模型的API，它能让原本普通的、只可以对话的ai长出双手，控制你的电脑，完成你给它的任务。最重要的是这一切完全是个性化、本地化、在线化。这是一种很“私人”的体验。

不过，它带来的安全问题也是我们要注意的，把自己的电脑全权交给一个ai，它是否值得我们信任？这就仁者见仁智者见智了。当然你也可以选择部署在云服务器上，这其实是最优解。但笔者并不想花这个钱，所以就选择本地部署了。

准备#

首先我们需要准备三样东西

  1. 一台电脑
  2. 一个大模型的ApiKey
    • 推荐免费的GLM-4.7-Flash（免费的额度够用很久）
  3. 耐心

环境依赖工具#

  1. 安装Node.js（官方与我都强烈推荐你在WSL2下部署OpenClaw）
    1. 在wsl2中安装nvm（如果是科学上网且用的是clash的话记得开始tun模式） 
      curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash

      export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"

      ## 测试版本号
nvm -v
## 安装nodejs
nvm install --lts
## 测试版本
node --version
      ✅ node.js版本 >= v22
  2. 安装Git
    在wsl2中运行 
    sudo apt-get install git
## 测试版本
git -v
  3. 安装OpenClaw 
    npm i -g openclaw
## 测试版本
openclaw --versiob

可以看到此时已经安装成功了 4. 初始化

openclaw onboard

快速入门#

开始初始化之后，它会让我们来进行设置，这里我会一步步讲解，你也可以直接抄我的作业

  1. 无视风险，继续安装。然后会有两个选项
    • QuickStart（快速开始）
    • Manual（手动）
  2. 这里我们选择QuickStart，之后就是就可以选择自己的模型，并输入APIKEY（把自己的KEY复制粘贴进来即可），这里以我的智谱为例（Z.AI）
  3. 模型我们选择4.7-flash
  4. 接入telegram机器人
    1. 参考官方文档
  6. 默认Telegram配置为白名单模式，不接收任何对话，需要我们手动开启白名单 
    openclaw config set channels.telegram.groupAllowFrom "<你的Telegram用户ID>"
  7. 配对，在telegram中向openclaw发送消息，获取配对码。 
    openclaw pairing approve telegram <code>
