定义规则，比实现更重要。

前言#

在上一章中，我们学习了“继承”和“多态”，让类与类之间能够共享结构、灵活协作。
但在实际开发中，我们往往还需要一种“模板”——它不关心具体实现，只规定所有子类必须遵守的行为。

Java 为此提供了两种机制：抽象类（abstract class） 与 接口（interface）。

抽象类定义“是什么”，接口定义“能做什么”；
它们让我们的代码既统一规范，又灵活可扩展。

本章，我们将深入理解抽象类与接口的区别与联系，
彻底掌握“设计规范”的艺术。

抽象类（Abstract Class）#

1. 为什么需要抽象类#

假设我们有一个动物类：

class Animal {
    void eat() {
        System.out.println("动物在吃东西");
    }
    void sound() {
        System.out.println("动物在叫");
    }
}

但问题是——并不是所有动物都一样叫。
有的汪汪，有的喵喵。
Animal类怎么写都不合适。

怎么办？
答案是：
让父类只定义“要叫”，但不写“怎么叫”。

2.抽象类的定义#

abstract class Animal {
    abstract void sound(); // 抽象方法
    void eat() {
        System.out.println("动物在吃饭");
    }
}

特点：

  • 使用 abstract 关键字；
  • 抽象方法没有方法体；
  • 含有抽象方法的类必须是抽象类；
  • 抽象类不能实例化（即不能 new Animal()）。

3. 抽象类的继承与实现#

子类必须实现父类的所有抽象方法：

class Dog extends Animal {
    void sound() {
        System.out.println("汪汪汪！");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("喵喵喵！");
    }
}

public class Test {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.sound(); // 汪汪汪！
        a2.sound(); // 喵喵喵！
    }
}

抽象类保证了每个“动物”都有 sound() 方法，
但让“怎么叫”交给子类去定义。

这就是“模板式”的设计思想。

4.抽象类的特点#

特点 说明
抽象类不能被实例化(不能被new) 只能作为父类使用
可以有构造方法 供子类 super() 调用
可以有成员变量、普通方法、抽象方法 不仅仅是“空壳”
子类必须实现所有抽象方法 否则子类也要是抽象类

接口( Interface)#

1. 接口的定义#

接口是一组规范。
它只定义“能做什么”，不管“怎么做”。

interface USB {
    void connect();
    void disconnect();
}

实现接口的类必须实现接口的所有方法：

class Mouse implements USB {
    public void connect() {
        System.out.println("鼠标连接成功");
    }
    public void disconnect() {
        System.out.println("鼠标已断开");
    }
}

class Keyboard implements USB {
    public void connect() {
        System.out.println("键盘连接成功");
    }
    public void disconnect() {
        System.out.println("键盘已断开");
    }
}

使用：

public class Test {
    public static void main(String[] args) {
        USB u1 = new Mouse();
        USB u2 = new Keyboard();

        u1.connect();
        u2.connect();
    }
}

输出：

鼠标连接成功  
键盘连接成功

2.接口的特点#

特性 说明
用 interface 定义 不是 class
所有方法默认是 public abstract 即使不写
所有变量默认是 public static final 相当于常量
接口不能实例化 只能由类 implements
一个类可以实现多个接口 ✅ 支持多继承效果

3.多接口的实现#

interface A { void run(); }
interface B { void jump(); }

class C implements A, B {
    public void run() { System.out.println("跑起来！"); }
    public void jump() { System.out.println("跳起来！"); }
}

接口弥补了 Java 不支持类多继承的缺点。
一个类可以同时遵守多个“规范”。

抽象类和接口的区别#

对比项 抽象类 接口
关键字 abstract class interface
是否可以有成员变量 ✅ 可以 ❌ 只有常量
是否可以有构造方法 ✅ 可以 ❌ 不行
是否可以多继承 ❌ 只能一个父类 ✅ 可实现多个接口
设计目的 代码复用 + 规范 纯规范约束

抽象类和接口的使用#

interface Flyable {
    void fly();
}

abstract class Animal {
    abstract void sound();
}

class Bird extends Animal implements Flyable {
    void sound() { System.out.println("叽叽喳喳"); }
    public void fly() { System.out.println("鸟在天空飞翔"); }
}

使用：

public class Test {
    public static void main(String[] args) {
        Bird b = new Bird();
        b.sound();
        b.fly();
    }
}

输出：

叽叽喳喳  
鸟在天空飞翔

抽象类和接口的实例#

interface Soundable {
    void makeSound();
}

abstract class Animal {
    String name;
    abstract void move();
}

class Dog extends Animal implements Soundable {
    void move() {
        System.out.println(name + " 在跑");
    }
    public void makeSound() {
        System.out.println(name + "：汪汪！");
    }
}

class Bird extends Animal implements Soundable {
    void move() {
        System.out.println(name + " 在飞");
    }
    public void makeSound() {
        System.out.println(name + "：啾啾！");
    }
}

public class Test {
    public static void main(String[] args) {
        Soundable[] arr = { new Dog(), new Bird() };
        arr[0].makeSound();
        arr[1].makeSound();
    }
}

输出：

汪汪！
啾啾！

接口让程序具备了“多态的终极自由”：
不管是谁，只要“会叫”，我就能调用它的 makeSound()。
