前言#

没啥好说的，Java中简单却又不可缺少的一部分，干就完了兄弟

包（Package）#

1.什么是包？#

包就像文件夹，用来存放类文件，防止命名冲突。
在 Java 中，类名可能重复，但包名不同就没问题。

定义包语法：

package 包名;

示例：

package com.school.student;

这个类的完整名字其实是：

com.school.student.Student

2.包的命名规则#

  • 一般使用 公司域名倒写 + 模块名
    • 例如：com.baidu.map、org.apache.commons
  • 全部小写
  • 用“.”分隔层级

3.包的使用#

src/    
 └─ com/    
     └─ demo/   
         ├─ Student.java    
         └─ Test.java

Student.java：

package com.demo;
public class Student {
    public void study() {
        System.out.println("学生在学习");
    }
}

Test.java：

package com.demo;
public class Test {
    public static void main(String[] args) {
        Student s = new Student();
        s.study();
    }
}

因为它们在同一个包下，所以可以直接访问。

4.导入其他包中的类#

当类不在同一个包中，需要 import。

import 包名.类名;

示例：

import java.util.Scanner;

你也可以导入整个包

import java.util.*;

但要注意，* 只会导入包下的类，不包括子包。

访问控制（Access Control）#

Java 提供了 4 种访问修饰符，用来控制谁能访问谁。

修饰符 同类 同包 子类 其他包
public ✅ ✅ ✅ ✅
protected ✅ ✅ ✅ ❌
(默认) ✅ ✅ ❌ ❌
private ✅ ❌ ❌ ❌

public（公共的）#

最开放的访问级别，任何类都能访问。

public class Demo { }

常用于对外提供的工具类、接口、公共方法等。

2. private（私有的）#

仅限当前类使用，外部无法访问。

class Student {
    private int age;

    public void setAge(int a) {
        if (a >= 0) age = a;
    }

    public int getAge() {
        return age;
    }
}

这样外部就不能随便改 age，必须通过方法访问。

这就是“封装（Encapsulation）”思想。

3. protected（受保护的）#

可在同包或子类中访问。

class Animal {
    protected void eat() {
        System.out.println("动物在吃东西");
    }
}

class Dog extends Animal {
    void test() {
        eat(); // ✅ 子类可访问
    }
}

4. 默认（包访问级别）#

如果不写任何修饰符，表示“包内可见”：

class Teacher {
    void teach() {
        System.out.println("老师在讲课");
    }
}

在同包下可以访问，跨包就不行。

类的可见性#

在 Java 中，一个 .java 文件中最多只能有一个 public 类。
并且它的名字必须与文件名相同。

// 文件名：Student.java
public class Student { }

其他类可以是默认访问级别（包内可见）。

包与访问控制的综合理解#

类位置 成员修饰符 是否可访问 示例
同包 默认 ✅ 直接调用
不同包 默认 ❌ 需改成 public
子类 protected ✅ 可继承访问
非子类 protected ❌ 不可访问
同类 private ✅ 自己内部访问
