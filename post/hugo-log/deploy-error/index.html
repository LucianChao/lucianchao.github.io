【建站日志|异常解决】hugo自动部署异常溯源过程及解决办法

2025-12-11 · (最后修改于 2025-12-13) · 2 分钟
目录
  • 起因
  • 溯源过程
  • 解决方案

起因#

昨天写完文章推送至远程仓库之后，今天发现网页没有更新，将问题溯源过程以及解决办法进行总结

溯源过程#

先重新推送一遍文章到远程仓库

网页端一开始并没有更新，这在以往是很正常的，因为我的站点用到了edgeone的cdn加速，需要等待cdn缓存刷新。
过了半小时左右，我再次打开网站，依旧没有更新，此时我意识到事情有些不对。

先说一下我的自动部署方案，我有两个仓库，A仓库是一个私有仓库，负责存放整个hugo博客资源。B仓库是一个公开的仓库，也是这个博客的githubPages，负责存放hugo生成的静态网页资源。
当我在本地写完文章，用git提交到A仓库时，触发github actions，运行我的workflow，生成网页资源到B仓库。

于是我进入A仓库查看github actions的记录，果然发现最近两条推送都报错。

查看报错信息 

/usr/bin/git push origin main
  remote: Invalid username or token. Password authentication is not supported for Git operations.
  fatal: Authentication failed for 'https://github.com/LucianChao/lucianchao.github.io.git/'
  Error: Action failed with "The process '/usr/bin/git' failed with exit code 128"

查看workflows脚本

name: deploy

# 代码提交到main分支时触发github action
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
        - name: Checkout
          uses: actions/checkout@v4
          with:
              fetch-depth: 0

        - name: Setup Hugo
          uses: peaceiris/actions-hugo@v3
          with:
              hugo-version: "latest"
              extended: true

        - name: Build Web
          run: hugo -D --ignoreCache --cleanDestinationDir

        - name: Deploy Web
          uses: peaceiris/actions-gh-pages@v4
          with:
              PERSONAL_TOKEN: ${{ secrets.TOKEN }}
              EXTERNAL_REPOSITORY: LucianChao/lucianchao.github.io
              PUBLISH_BRANCH: main
              PUBLISH_DIR: ./public
              commit_message: auto deploy

分析得知是 认证失败（Invalid username or token）,也就是workflow中的${{ secrets.TOKEN }}失效

              PERSONAL_TOKEN: ${{ secrets.TOKEN }}

这让我想起昨晚我的Github Tokens失效，重新生成后直接提交了一次，提交成功，但今天站点并没有更新。
那么现在问题应该已经很明了了：重新生成的token并没有同步配置到仓库中，导致仓库A没有权限访问仓库B，也就无法部署静态资源到仓库B

解决方案#

  1. 把新的token配置到仓库中
    Settings->Secrets and variables->Actions
  2. 重新部署：
    • 重新提交一次
    • 把失败的部署重新运行
  3. 查看博客站点，已经成功更新
