前言#

警告

血与泪的教训，永远不要让Fancybox处理大文件

目前常见的图片放大方案有这几种

方案 优点 适合场景
Medium Zoom 极简、轻量 普通文章、单图放大
Zooming.js 极简、轻量、可拖动 技术博客
lightGallery 图集、轮播 图多、文章图集、相册
Fancybox UI 美观、多媒体 图片+视频+iframe混合内容

几种方案我都试用了一下，阐述一下个人感受：

  • Medium Zoom和Zooming.js：很极简，没有任何多余ui，点击放大，说实话很对我的胃口。但移动端的体验绝对称不上美妙，不支持捏合缩放，没有手势优化，可以说很糟糕了。
  • Fancybox —— 功能最全面的弹窗方案，支持捏合缩放、多图滑动、图片标题和分组展示。

综合下来，为了在移动端能有更好的体验，放弃了极简设计，转而使用了功能更丰富的Fancybox。

功能实现#

  1. 新建layouts\_default\_markup\render-image.html 
    {{/* 先声明变量 $src */}}
{{ $src := "" }}

{{/* 获取 Page Bundle 图片资源 */}}
{{ $res := .Page.Resources.GetMatch (path.Base .Destination) }}
{{ if $res }}
  {{ $src = $res.RelPermalink }}  {{/* Page Bundle 内图片路径 */}}
{{ else }}
  {{ $src = .Destination | absURL }}  {{/* 普通路径 */}}
{{ end }}

{{ $alt := .Text }}
{{ $title := .Title }}
{{ $thumbnail := .Page.Params.thumbnail | default $src }}

<div class="post-img-view">
  <a
    data-fancybox="post-{{ .Page.File.Path }}"
    href="{{ $src }}"
    data-caption="{{ $title }}"
  >
    <img
      src="{{ $thumbnail }}"
      srcset="{{ $src }} 2x"
      alt="{{ $alt }}"
      loading="lazy"
      style="max-width: 100%; height: auto"
    />
  </a>
</div>

<!-- 
$src        ：原图地址（Page Bundle 自动解析）
$thumbnail  ：缩略图，如果没有则用原图
data-fancybox：每篇文章独立分组 gallery
data-caption：图片描述
srcset      ：支持 Retina 高清显示
loading     ：延迟加载
样式        ：最大宽度100%，高度自适应
-->
  2. 在 layouts/partials 下的 extend_head.html 或者 extend_footer.html 中添加 
    <!-- Fancybox -->
{{ if .IsPage }}
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@fancyapps/ui/dist/fancybox.css"
/>
<script src="https://cdn.jsdelivr.net/npm/@fancyapps/ui/dist/fancybox.umd.js"></script>
<script>
  document.addEventListener("DOMContentLoaded", function () {
    Fancybox.bind("[data-fancybox]", {// 绑定所有 data-fancybox 图片
      animated: true,                  // 启用动画效果
      showClass: 'fancybox-fadeIn',    // 打开动画
      hideClass: 'fancybox-fadeOut',   // 关闭动画
      dragToClose: true                 // 启用拖拽关闭
    });
  });
</script>
{{ end }}

效果预览#

语法：

![测试图片](图片链接 "图片描述")

效果预览：

支持左右滑动切换图片，上下拖动关闭，也支持Page Bundle 功能（相对路径的资源引用）
