前言#

前几天给博客上线了音乐播放器功能，技术选型是 APlayer + MetingJS，并尝试配合 PJAX / Barba.js 实现无刷新页面切换。 一开始在本地测试时一切正常：

  • 播放器能正常加载
  • 音乐可以播放
  • 打开新页面之后播放器后能保持播放

但上线并实际使用后，问题开始集中爆发，而且问题之间相互叠加，极难定位，包括但不限于：

  • ❌ 页面切换后播放器消失 / 重复初始化
  • ❌ 主页样式异常
  • ❌ 歌词不显示或偶发显示
  • ❌ 手机端访问无播放器
  • ❌ 控制台无明显报错，但行为异常
  • ❌ 与主题（PaperMod）脚本强耦合
  • ❌ 博客原本功能异常
    • ❌ 代码块复制按钮消失
    • ❌ fancybox图片浏览功能异常（打开图片后直接跳转到新页面）
    • ❌ 搜索功能失效
    • 等等等等……（持续排查中）

可以说，这是一次 前端架构层面的灾难性设计失误。 这篇文章记录完整的排查过程、踩坑原因，以及最终的修复方案。

技术背景 & 初始方案#

技术栈#

  • 博客框架：Hugo
  • 主题：PaperMod
  • 音乐播放器：
    • APlayer v1.10.1
    • MetingJS v2.0.2 页面切换：
    • PJAX（后期尝试 Barba.js）

最初设计思路#

最初的想法非常理想化
先是只引入 MetingJS ，此时一切正常，播放器能加载、能播放、歌词也正常地显示。

功能一旦实现，人就开始追求体验了。
于是 我就开始折腾了😅，
我试图实现 页面切换不刷新播放器 来让 歌曲持续播放 时 ，又引入了Barba.js（一个高级Pjax框架），它通过拦截页面跳转，实现“伪SPA”效果

到这里，问题其实已经不是“是否会出问题”，
而是什么时候彻底炸。
框架本身没毛，它确实是一个好框架，设计的很优秀。是我太蠢了。

我忽略了生命周期：
我默认“页面切换 ≈ 页面重新加载”
却忽略了在 PJAX / Barba 场景下：

  • DOM 会被替换，但页面并不会真正刷新
  • DOMContentLoaded 只会触发一次

代价是惨痛的。
结果是：

播放器以为自己是全局单例
主题以为页面永远是新加载的

博客原本功能、自定义功能的生命周期全部乱套。
当这些假设互相冲突时，
问题就不再是“某个功能失效”，
而是整个页面行为开始变得不可预测。

页面开始随即坏
功能就像量子态
你不看它，它就不坏

原因分析#

写这篇文章时，这些bug已经被修复的七七八八了。
回头看，根本原因就是：太低估这些框架被引入之后，系统的复杂度

  3. 这里我们对aplayer和metingjs做了整合，把整个过程封装成了函数（解耦），然后在页面第一次加载时调用一次函数，在这个函数的内部又做了逻辑处理，也就是第二次校验，当播放器已经被创建时，也不会再次创建播放器对象。 
    console.log("player.js loaded");

(function () {
  async function initPlayerOnce() {

    if (window.__aplayer_inited__) {
      console.log("APlayer already inited, skip");
      return;
    }

    const container = document.getElementById("aplayer");
    if (!container) {
      console.warn("APlayer container not found");
      return;
    }

    /* ======================
       1. 自托管 FLAC
       ====================== */
    const localAudio = [
      //   {
      //     name: 'Interstellar',
      //     artist: 'Hans Zimmer',
      //     url: '/music/flac/interstellar.flac',
      //     cover: '/music/cover/interstellar.jpg',
      //     lrc: '/music/lrc/interstellar.lrc',
      //     type: 'flac'
      //   }
    ];

    /* ======================
       2. Meting（网易云平台）
       ====================== */
    let metingAudio = [];
    try {
      const res = await fetch(
        "https://api.i-meto.com/meting/api?server=netease&type=playlist&id=9729906436"
      );
      metingAudio = await res.json();
    } catch (e) {
      console.warn("Meting load failed, fallback to local only");
    }

    /* ======================
       3. 合并 & 创建 APlayer
       ====================== */
    const audioList = [...localAudio, ...metingAudio];

    window.aplayer = new APlayer({
      container,
      fixed: true,
      autoplay: true,
      preload: "auto",
      lrcType: 1,
      //   audio: audioList,
      audio: metingAudio,
    });
    window.__aplayer_inited__ = true;
    console.log("APlayer created with", audioList.length, "tracks");
  }

  document.addEventListener("DOMContentLoaded", initPlayerOnce);
  // PJAX 切页完成后（Barba）
  document.addEventListener("barba:afterEnter", () => {
    // 播放器不参与 PJAX
  });
})();

2️⃣fancybox失效（跳转到浏览器默认查看图片页面）#

点击图片之后，本应该调用fancybox来查看图片，但是fancybox界面出现之后，又跳转到了新页面。
所以可以知道的是，fancybox加载了也初始化了，但是可能被阻止或者覆盖。

  1. 然后我们去修改fancybox.html中的脚本 
    <link
  rel="stylesheet"
  href="/lib/fancybox/fancybox.css"
/>
<script src="/lib/fancybox/fancybox.umd.js"></script>
<script>
   function initFancybox() {
    const images = document.querySelectorAll("[data-fancybox]");
    if (!images.length) return;

    Fancybox.bind("[data-fancybox]", {
      animated: true,
      showClass: "fancybox-fadeIn",
      hideClass: "fancybox-fadeOut",
      dragToClose: true,
    });
  }

 //阻止 a 的默认跳转
  document.addEventListener("click", function (e) {
    const link = e.target.closest("a[data-fancybox]");
    if (!link) return;

    e.preventDefault();  //阻止 <a href> 的原生跳转
    e.stopPropagation(); // 防止被 PJAX / 其他监听吃掉
  });

  // 初始化
  if (document.readyState !== "loading") {
    initFancybox();
  } else {
    document.addEventListener("DOMContentLoaded", initFancybox);
  }
  // PJAX 切页后
  document.addEventListener("barba:afterEnter", initFancybox);
</script>
    在现在的逻辑中，我们对点击事件做了阻止，来试图阻止浏览器pjax吃掉点击事件，然后封装成函数，再barba初始化时调用，切换页面时再次调用。来让每个页面都能有自己的fancybox。
  2. 此时我们点击图片，可以正常打开，但是切换到新页面之后，会发现，fancybox还是失效。
    再次进行修改 
    <link rel="stylesheet" href="/lib/fancybox/fancybox.css" />
<script src="/lib/fancybox/fancybox.umd.js"></script>
<script>
  function initFancybox() {
    Fancybox.unbind("[data-fancybox]");

    Fancybox.bind("[data-fancybox]", {
      animated: true,
      showClass: "fancybox-fadeIn",
      hideClass: "fancybox-fadeOut",
      dragToClose: true,
    });
  }

  // 初始化
  if (document.readyState !== "loading") {
    initFancybox();
  } else {
    document.addEventListener("DOMContentLoaded", initFancybox);
  }
  // PJAX 切页后
  document.addEventListener("barba:afterEnter", initFancybox);
</script>
    删掉了对<a>超链接跳转的阻止，因为fancybox会在click时阻止默认行为，只要是有[data-fancybox]属性的元素都会进行处理。
