错误代码#

无法将磁盘“C:\Users\QiaoZ\AppData\Local\Packages\CanonicalGroupLimited.Ubuntu_79rhkp1fndgsc\LocalState\ext4.vhdx”附加到 WSL2: 系统找不到指定的文件。
错误代码: Wsl/Service/CreateInstance/MountDisk/HCS/ERROR_FILE_NOT_FOUND

具体原因分析#

WSL2 尝试启动 Ubuntu 实例时，找不到对应的虚拟硬盘文件 ext4.vhdx。

ext4.vhdx 就是 Ubuntu 的整个文件系统（相当于一个虚拟硬盘）。
出现这个报错 98% 的可能性就是 ext4.vhdx被意外删除、移动，或者被杀毒软件隔离，WSL 就无法挂载它，自然启动失败。

常见的情况就是误删（清理C盘文件）。

但也有一个比较坑的可能性 1，就是windows更新。因为笔者在遇到这个情况的前几天刚刚更新过

解决办法#

  1. 查看当前WSL实例 
    wsl --list --verbose
  2. 重置实例 
    wsl --unregister Ubuntu
  3. 重新安装Ubuntu（通过Microsoft Store / PowerShell ） 
    wsl --install -d Ubuntu
  4. 下载成功后会自动安装，第一次启动时会创建默认Unix用户，需要设置用户名和密码。

命令推荐#

mkdir E:\WSL\backup
# 导出（相当于拍快照\ 路径可改）
wsl --export Ubuntu E:\WSL\backup\ubuntu_backup.tar
# 回滚 / 恢复
wsl --unregister Ubuntu
# wsl --import <名字/新名字> <安装目录> <备份tar路径>
wsl --import Ubuntu E:\WSL\Ubuntu E:\WSL\backup\ubuntu_backup.tar

  1. WSL not working after windows update(https://github.com/Microsoft/WSL/issues/13948) ↩︎
