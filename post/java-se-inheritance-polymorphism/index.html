到目前为止，我们已经能写出功能完整的程序，
使用类、对象、方法来组织逻辑，也学会了 static 与 final 的妙用。

但是，当项目越来越大时，你可能会发现：

  • 不同类之间有很多相似的代码；
  • 某些功能需要在不同类之间共享或稍作修改；
  • 想要让程序更灵活，却发现改一处要动全身。

于是，Java 提供了两个强大的“法宝”：
继承（Inheritance） 和 多态（Polymorphism）。

继承让类之间能够建立“父与子”的关系，实现代码复用；
多态则让子类可以拥有自己的行为，却能在父类引用下“自由切换”。

本章，我们将一起揭开这两位“面向对象核心成员”的面纱，
让你的代码从“能用”变成“优雅”。

继承（Inheritance）#

站在巨人的肩膀上

1. 为什么需要继承#

假设我们有两个类：

class Dog {
    String name;
    void eat() {
        System.out.println(name + "正在吃饭");
    }
}

class Cat {
    String name;
    void eat() {
        System.out.println(name + "正在吃饭");
    }
}

Cat类与Dog类都能吃饭，代码重复。
有没有办法让它们共用同一份“吃饭逻辑”呢？
有的兄弟，有的——那就是"继承"。

2.继承的基本语法#

class 父类 {
    // 父类成员
}

class 子类 extends 父类 {
    // 子类成员
}

关键字 extends 表示“继承自”。 子类会自动拥有父类的所有非私有成员。

改写上面的例子：

class Animal {
    String name;
    void eat() {
        System.out.println(name + "正在吃饭");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println(name + "在汪汪叫");
    }
}

class Cat extends Animal {
    void meow() {
        System.out.println(name + "在喵喵叫");
    }
}

使用：

public class Test {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.name = "旺财";
        d.eat();  // 继承自Animal
        d.bark(); // 自己的
    }
}

输出：

旺财正在吃饭  
旺财在汪汪叫

3.继承的特点#

特性 说明
单继承 Java 中一个类只能继承一个父类
多层继承 子类的子类也可以继续继承
构造函数不被继承 但会自动调用父类构造函数
private 成员不会被继承 但可以通过 public 或 protected 方法间接访问

4.super：子类与父类的桥梁#

super 关键字用于访问父类的成员。

super有三种用法：

  1. 调用父类的成员变量： 
     super.name
  2. 调用父类的方法： 
    super.eat();
  3. 调用父类的构造方法： 
    super(参数);

信息

super() 必须放在子类构造函数的第一行。
子类全部构造器写不写都有，会自动调用父类的无参构造器

“如果父类没有无参构造器，而子类又没有显式调用 super(参数)，编译会报错。”

5.方法重写（Override）#

子类可以重写父类的方法，用自己的实现替换父类逻辑。

class Animal {
    void eat() {
        System.out.println("动物在吃东西");
    }
}

class Dog extends Animal {
    @Override
    void eat() {
        System.out.println("狗在啃骨头");
    }
}

运行：

Dog d = new Dog();
d.eat(); // 狗在啃骨头

提示

  • @Override 是注解，用来提醒编译器“这确实是一个重写”。
  • 重写时，方法名、参数、返回值类型必须完全一致。
  • 子类访问修饰符不能比父类更严格（例如父类是 public，子类不能是 private）。

多态( Polymorphism)#

多态的核心思想是：

“父类引用可以指向子类对象。”

Animal a = new Dog();
a.eat();  // 输出：狗在啃骨头

虽然 a 的类型是 Animal，但它实际上引用的是 Dog。
在运行时，会调用 Dog 的版本。

这就是 运行时多态。

1.多态的三个必要条件#

条件 说明
继承 子类继承父类
重写 子类重写父类方法
向上转型 父类引用指向子类对象

2.多态的实例#

class Animal {
    void sound() {
        System.out.println("动物在叫");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("狗在汪汪叫");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("猫在喵喵叫");
    }
}

public class Test {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.sound(); // 狗在汪汪叫
        a2.sound(); // 猫在喵喵叫
    }
}

这样写的好处是：
我们可以轻松扩展更多动物，而不需要修改原代码。

3.向上转型与向下转型#

向上转型（Upcasting）#

子类对象 → 父类引用 ✅ 安全

Animal a = new Dog();

向下转型（Downcasting）#

父类引用 → 子类对象 ❗需要强制转换

Dog d = (Dog) a;

但一定要确保这个对象本来就是该类型，否则会抛出 ClassCastException。

因此，在转换前最好先判断：

if (a instanceof Dog) {
    ((Dog)a).bark();
}

4.final 与继承的关系#

  • final 修饰类 → 该类不能被继承
  • final 修饰方法 → 该方法不能被重写
final class Tool { } // ❌ 不能有子类

继承与多态的应用#

场景 示例 说明
图形系统 父类 Shape，子类 Circle、Rectangle 统一调用 draw()
游戏角色 父类 Role，子类 Warrior、Mage 共享攻击逻辑，独立技能
GUI 组件 父类 Component，子类 Button、TextField 通用渲染接口
